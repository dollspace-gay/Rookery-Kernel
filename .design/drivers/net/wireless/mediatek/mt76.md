## Summary

mt76 is the MediaTek WiFi driver family — covers MediaTek WiFi silicon from mt76x0 (legacy 802.11n) through mt7996 (current 802.11be / WiFi 7). MediaTek is the second-largest WiFi silicon supplier after Qualcomm globally; in the consumer router market it is the most-deployed because Qualcomm's IPQ silicon dominates the high-end AP market while MediaTek dominates the mid-/low-end consumer market. Every cheap TP-Link Archer, every Asus low-mid range router, every Xiaomi router, every cheap Chinese router on AliExpress, every OpenWrt-targeted small AP runs MediaTek silicon. Also: mt7921 + mt7925 are popular in cheap laptops (Acer, HP Pavilion, Lenovo IdeaPad) as the WiFi card.

The driver is structured as a common core (`mt76*.c` at the top level) + per-silicon-family subdirectories. The Connac platform abstraction (mt76_connac, mt76_connac3) is the modern shared layer; older silicon (mt76x0, mt76x2, mt7603, mt7615) has its own per-family code; newest silicon (mt7921, mt7925, mt7996) shares through mt76_connac{,3}.

Source: ~95500 lines / many .c files across the per-silicon subdirectories. Larger than iwlwifi or ath11k in total because of the silicon-zoo span (2011 mt76x0 through 2024 mt7996).

This Tier-3 covers mt76.

## Rust translation posture

mt76 has the per-silicon-subdir explosion characteristic of vendor drivers covering many generations. Translation:

- **`mt76-core` crate** — mac80211 integration, MCU cmd transport common, DMA descriptor common, A-MPDU reorder, EEPROM, channel mgmt.
- **`mt76-connac` crate** — shared platform layer for mt7615+ + mt7921+. Connac MCU + Connac MAC common.
- **`mt76-connac3` crate** — newest Connac iteration for mt7996+.
- **Per-silicon-family crates**: `mt76x0`, `mt76x2`, `mt7603`, `mt7615`, `mt7915`, `mt7921`, `mt7925`, `mt7996`. Each implements per-silicon mac80211 ops + per-silicon MCU cmd builders.
- **`Mt76Dev<Phase>`** typed lifecycle.
- **MCU cmd protocol typed.** MediaTek's MCU = on-chip microcontroller that runs the WiFi firmware; host cmds via MCU mailbox.
- **Per-bus transport.** USB + PCIe + SDIO + AHB (SoC-integrated) — different mt76 subdrivers per silicon use different transports.

The grsec/PaX section is mandatory and parallels iwlwifi/ath11k. Historical CVE concentration in MCU cmd parsing + A-MPDU reorder (sequence-number race bugs).

## Compatibility contract

REQ-1: PCI ID + USB ID + SoC-compatible table — every MediaTek WiFi silicon supported by upstream mt76. Frozen.

REQ-2: Per-silicon-family driver-binding — at probe, dispatch to right subdir's probe based on PCI/USB/SoC ID.

REQ-3: MCU cmd protocol — per-silicon variant; Connac silicon shares Connac MCU; legacy silicon (mt76x0/x2/7603/7615) has its own.

REQ-4: mac80211 integration — full ieee80211_ops at the mt76-core level + per-silicon ops table.

REQ-5: Per-PHY abstraction — multi-PHY silicon (mt7915/7916/7996 have separate 2.4G + 5G + 6G PHYs) expose each PHY as a separate mac80211 wiphy.

REQ-6: WiFi 4/5/6/6E/7 features per silicon — full feature stack per silicon spec.

REQ-7: DFS — radar detection mandatory on DFS channels (5 GHz); per-region radar pattern database.

REQ-8: SAR (Specific Absorption Rate) — per-band tx power limits per regulatory + ACPI tables.

REQ-9: Power save — PSM, U-APSD, TWT (WiFi 6+).

REQ-10: AP-mode — supported on most silicon; used in TP-Link/Asus/Xiaomi consumer routers.

REQ-11: Multi-bus — PCIe (laptops, discrete cards), USB (older silicon + USB dongles), SDIO (some embedded), AHB (SoC-integrated APs).

REQ-12: FW images — MediaTek-signed FW per silicon (different image per silicon family).

REQ-13: TXOP scheduler (per-silicon).

REQ-14: Mesh + EasyMesh (newer silicon) — for multi-AP mesh systems (Asus AiMesh, TP-Link OneMesh, Xiaomi Mesh).

REQ-15: A-MPDU + A-MSDU.

REQ-16: WPA2 + WPA3 + 802.1X.

## Architecture

**Core + Connac + per-silicon layering.** mt76-core owns mac80211 integration + DMA + MCU transport. mt76_connac{,3} owns shared platform layer for the modern silicon families. Each per-silicon subdir owns its specific MAC + MCU cmd implementations.

**Per-silicon MCU cmds.** Each silicon family has its own MCU cmd protocol (because each generation's FW evolves). Connac silicon shares Connac cmd format; pre-Connac silicon (mt76x0/x2/7603/7615) has its own. Cmds typed per-silicon crate.

**Per-PHY abstraction.** Multi-PHY silicon (mt7915, mt7996) has separate 2.4G + 5G (+6G on 7996) PHYs. Each registered as a separate wiphy with mac80211. Per-PHY state (channel, beacon, STA list).

**A-MPDU reorder.** Per-TID per-STA reorder buffer. Sequence-number-window mgmt at receive. Edge cases at window-shift + retransmits. Historical bug area.

**Multi-bus transport.** USB transport (mt76x0 dongles, mt76x2u): USB-bulk IN/OUT endpoints for MCU + RX/TX. PCIe transport: standard MMIO + IRQ + DMA rings. SDIO (some embedded). AHB (SoC-integrated APs).

**DFS.** Mandatory radar detection on DFS channels (5 GHz 5250-5350 + 5470-5725 MHz). Per-region radar pattern database; on radar event, vacate channel within 10s.

**SAR.** Per-band tx-power limit derived from ACPI WGDS/WRDS/WTDS tables (mt76 reuses Intel's table format for some platforms) + per-regulatory.

**FW image + signed.** MediaTek-signed FW. Signature checked by silicon secure-boot.

## Grsecurity/PaX-style Reinforcement

This driver inherits the standard PaX/Grsecurity surface and reinforces it with:

- **PAX_USERCOPY** — debugfs writes, sysfs writes, ACPI SAR table parse all bounded.
- **PAX_KERNEXEC** — core ops, McuOps trait dispatch, per-silicon op tables, mac80211 ops, DMA ops, A-MPDU reorder dispatcher all `const` after init in W^X memory.
- **PAX_RANDKSTACK** — nl80211 / wireless-ext / debugfs entry paths per-syscall stack-rerandomized.
- **PAX_REFCOUNT** — `mt76_dev` refcount, per-Phy refcount, per-wcid refcount, per-queue refcount, MCU-in-flight refcount all saturating.
- **PAX_MEMORY_SANITIZE** — Crypto key material `kfree_sensitive`. FW image load buffer cleared. MCU cmd DMA buffers cleared. SAR table parse scratch cleared.
- **PAX_UDEREF** — SMAP/SMEP on every user-facing entry.
- **PAX_RAP / kCFI** — core/per-silicon op tables, McuOps trait dispatch, MAC ops, DMA ops, A-MPDU dispatcher all kCFI-signed.
- **GRKERNSEC_HIDESYM** — debugfs (`/sys/kernel/debug/ieee80211/phy<N>/mt76/*`) scrubbed for non-root.
- **GRKERNSEC_DMESG** — verbose mt76 logs (per-MCU-cmd trace, A-MPDU reorder window, scan results) restricted from non-root.
- **CAP_NET_ADMIN required** — every state-changing nl80211 / wireless-ext op.
- **RF-layer trust boundary explicit** — anything in RF range is adversarial.
- **Per-frame-type RX parser strict** — beacon/probe-response/auth/action frames typed parsed bounded length.
- **FW image MediaTek-signature mandatory** — per silicon secure-boot enforces; LOCKDOWN_INTEGRITY_MAX disables FW updates.
- **MCU cmd opcode allowlist per silicon** — only known opcodes for this silicon's FW version accepted.
- **MCU response length validated per opcode** — closes parser-OOB.
- **A-MPDU sequence-number window strict** — reorder buffer size bounded, window-shift validated, BAR (Block Ack Request) source-STA validated. Closes 'malformed BA forces window shift → OOB write' class. CVE-class historical.
- **DFS radar-event validated** — radar event payload sanity-checked before vacating; closes 'rogue MCU forges radar event → DoS via channel-vacate' class.
- **SAR ACPI table parser bounded + validated** — closes 'BIOS exposes malformed SAR → driver UB' class.
- **Per-PHY tenant isolation** — when multi-PHY silicon serves multiple wiphy instances (per-band APs), per-PHY state strictly isolated; closes cross-PHY contamination.
- **USB transport length validation** — for USB-attached mt76x0/x2u, every USB bulk transfer's length validated against MAX_LENGTH constants. Closes 'malformed USB bulk packet from rogue dongle → driver OOB' class.
- **AP-mode beacon template safety** — beacon template cannot include vendor-extension fields that could leak host info (same posture as ath11k).
- **TKIP/WEP refused in WPA3-only** — same as other WiFi drivers.
- **CSA action frame source-MAC validation** — same.
- **EasyMesh / Mesh peer validation** — mesh peers must complete authentication before mesh-traffic accepted.
- **Regulatory ACPI validation** — same as Intel/Qualcomm WiFi.
- **FW coredump CAP_SYS_ADMIN** — gated.
- **MCU watchdog rate-limit** — MCU exception → FW reload rate-limited; closes 'compromised FW gaslights driver into endless reload' class.

Per-doc rationale: mt76 completes WiFi vendor coverage with the second consumer-router-dominant silicon family (alongside Realtek's RTL8125+ and Qualcomm's IPQ for high-end). The trust posture is identical (RF-layer adversarial) with parallel defenses. Unique to mt76: multi-bus zoo (USB + PCIe + SDIO + AHB), large per-silicon-family span (mt76x0 2011 → mt7996 2024), Connac shared-platform layer for modern silicon. USB-transport length validation is unique among kernel WiFi drivers (USB is a different attack vector vs PCIe — a malicious USB dongle could be plugged in by a physical-access attacker). Multi-PHY isolation matters for AP-mode silicon (mt7915/7996) where one PHY may serve guests and another internal traffic.

## Open Questions

- [ ] Q1: USB transport — physical-access threat model differs from PCIe (cf the BadUSB class). Should USB-mode mt76x0/x2u require explicit `CONFIG_USB_*` opt-in?
- [ ] Q2: Per-silicon family explosion — 8+ family crates. Maintenance burden. Convention: each as a separate Rust crate that can be Kconfig-disabled.
- [ ] Q3: Connac3 vs Connac2 vs pre-Connac — clean architectural boundary in Rust trait hierarchy.
- [ ] Q4: EasyMesh / AiMesh / OneMesh — vendor-specific mesh protocols on top of 802.11s. Defer to userspace tooling.
- [ ] Q5: WiFi 7 mt7996 — newer silicon, less mature. Validate the MLO + 4096-QAM paths.

## Verification

- **Kani SAFETY**: prove per-silicon family dispatch exhaustive. Prove A-MPDU reorder window operations bounded.
- **TLA+**: model A-MPDU reorder with concurrent RX + BAR + window-shift; check no OOB write under any sequence.
- **Verus**: functional spec of MCU cmd encoder + response parser — encode/decode roundtrip.
- **Kani+Verus**: invariant that every active wcid has tracked keys; every multi-PHY instance has tracked state; cleanup at unbind removes all.
- **Integration**: mt7921 laptop connect, mt7915 AP mode 100-STA stress, mt7996 WiFi 7 connection, mt76x0 USB dongle smoke, DFS channel switch, SAR limits, FW crash recovery.
- **Fuzz**: beacon/probe-response/auth/action frame fuzzer; MCU response fuzzer; A-MPDU BAR fuzzer; ACPI SAR table fuzzer; USB bulk packet fuzzer.
- **Penetration**: rogue AP malformed beacon — parser must refuse. Forged radar event from compromised FW — driver must validate before vacating channel. Malicious USB dongle — refuse beyond device-class limits.
