## Summary

r8169 is the Realtek Gigabit and 2.5 Gigabit Ethernet driver — covers the entire Realtek consumer/SMB NIC silicon family from RTL8169 (the original 2002 silicon that put GbE in every consumer board) through RTL8101/8102/8103/8105/8106/8107/8125/8126 (current 2.5G). This is statistically the most-deployed NIC silicon family on Earth — every consumer/prosumer motherboard, every cheap USB Ethernet adapter, every set-top box, every smart TV, every entry-level NAS, every Raspberry-Pi-class SBC. Hundreds of millions of units in production.

The driver lives across very many chip revisions: the upstream `rtl_chip_info[]` table contains 60+ distinct RTL chip versions, each potentially needing its own PHY init sequence + per-version quirks. The driver auto-detects chip version at probe (`__rtl8169_get_mac_version`) and dispatches to the right per-version setup. This per-chip variant zoo is the dominant architectural reality.

Source: ~7700 lines / 4 .c files for r8169 (plus rtase for the dedicated RTL8125 server-grade variant).

This Tier-3 covers r8169 main + phy_config + firmware + leds.

## Rust translation posture

r8169 has a very different shape from Intel/Mellanox NICs: tiny code surface, no SR-IOV, no DCB, no FCoE, no IPsec, no admin queue. The complexity is in the per-chip-version dispatch — 60+ distinct silicon revisions each with subtly different init sequences. Translation:

- **`r8169-core` crate** — netdev, NAPI, RX/TX, ethtool, link mgmt.
- **`ChipVersion` enum** — 60+ variants enumerating the rtl_chip_info[] table. Per-version trait `RtlChip` providing init/PHY/quirk methods.
- **`R8169Adapter<Phase>` typed lifecycle.**
- **PHY init sequences** as typed data tables per chip version (no executable code per-version — pure data).
- **Optional firmware loading** typed: per-chip firmware path lookup; signature check by FW interface.

Grsec is mandatory but lighter than enterprise NICs (no SR-IOV, no privileged passthru ioctl). Historical CVE concentration in the per-chip-version handling (CVE-class around wrong-chip-version detection writing wrong PHY regs).

## Compatibility contract

REQ-1: PCI ID table — RTL8169 + RTL8168(A-K) + RTL8101(B-J) + RTL8102 + RTL8103 + RTL8105 + RTL8106 + RTL8107 + RTL8125(A-D) + RTL8126. Frozen.

REQ-2: Chip-version detection — at probe, read MAC version register and dispatch via `rtl_chip_info[]` table to the matching per-version setup. 60+ variants supported.

REQ-3: Per-version PHY init — each chip version has its own PHY init sequence applied at probe + after PHY reset. Sequences are data tables of (reg, mask, val).

REQ-4: Optional firmware — some newer chip revs (RTL8125 onwards typically) need a small firmware blob loaded via `request_firmware` for PHY patches. Frozen against `MODULE_FIRMWARE` declarations.

REQ-5: Per-chip feature gating — some chips support hw-checksum-v2, USO (Unified Send Offload, IPv4+v6 TSO), large-send-v3, etc. Per-chip `supports_*` flags gate features.

REQ-6: Link auto-negotiation — 10/100/1000/2500 Mbps (per-chip-version max). EEE supported on some chip revs.

REQ-7: Standard netdev features — TSO, GSO, GRO, RX/TX checksum (per-chip), VLAN HW filter.

REQ-8: Wake-on-LAN — magic packet, link change.

REQ-9: LED config via Linux LED class (newer chips).

REQ-10: AER — pci_error_handlers.

REQ-11: NAPI per RX queue (typically 1 RX queue on consumer chips; some 8125 variants support more).

REQ-12: ASPM — PCIe ASPM L0s + L1; per-chip quirks (some chips need ASPM disabled for stability).

## Architecture

**Per-chip-version dispatch.** At probe, read MAC version register. `rtl_chip_info[]` array indexed by version-id, each entry containing: name, supported features, PHY init pointer, fw image path (optional), per-chip quirks. Driver dispatches via this table for init + feature gating.

**Standard netdev pattern.** Per-CPU NAPI (typically 1 queue on consumer chips). MSI-X-or-MSI-or-legacy-IRQ depending on chip version. Page-pool RX. Standard offload set.

**PHY init.** Per-version data table applied at probe + after PHY reset. Tables are dozens to hundreds of (reg, mask, val) tuples. Source-of-truth from Realtek; reverse-engineered + occasionally vendor-contributed.

**Optional firmware.** Newer chips (RTL8125+) load a small firmware blob (~50 KB) for PHY/MAC patches. Loaded via `request_firmware`. Realtek-signed in newer revs (kernel checks via firmware-signing infrastructure).

**Power mgmt.** Aggressive runtime PM — chip can enter D3 when link down + idle. Wake-on-LAN supported. ASPM L0s/L1 — but several chips have stability issues with ASPM enabled; per-chip quirks disable it.

## Hardening

- Chip-version detection cross-checked against secondary register reads; mismatch refuses to bind.
- PHY init sequence bounded by per-table length.
- Firmware blob signature checked.
- WOL setup capability-gated.

## Grsecurity/PaX-style Reinforcement

This driver inherits the standard PaX/Grsecurity surface and reinforces it with:

- **PAX_USERCOPY** — ethtool ingress bounded.
- **PAX_KERNEXEC** — netdev_ops, ethtool_ops, per-chip op table (rtl_chip_info[]), PHY init dispatcher, IRQ handler, AER handlers, LED ops all `const` after init in W^X memory.
- **PAX_RANDKSTACK** — ethtool / sysfs entry paths per-syscall stack-rerandomized.
- **PAX_REFCOUNT** — `rtl8169_private` refcount saturating.
- **PAX_MEMORY_SANITIZE** — Firmware load buffer cleared after load.
- **PAX_UDEREF** — SMAP/SMEP on every user-facing entry.
- **PAX_RAP / kCFI** — netdev_ops, ethtool_ops, per-chip dispatch, PHY init function pointers, IRQ handlers, AER handlers, LED ops all kCFI-signed.
- **GRKERNSEC_HIDESYM** — debugfs scrubbed.
- **GRKERNSEC_DMESG** — verbose r8169 logs restricted.
- **CAP_NET_ADMIN required** — every state-changing op: MAC change, RSS config, WOL setup, ethtool ring/coalesce.
- **Chip-version detection cross-checked** — primary version reg + secondary register-read cross-check; mismatch refuses to bind. Closes "spoofed chip-id triggers wrong PHY init sequence → brick PHY" class.
- **PHY init data tables immutable** — per-version tables are const data; cannot be modified at runtime. Closes a path where a malicious module could alter PHY init to brick.
- **Firmware blob signature mandatory (newer chips)** — kernel firmware-signing infrastructure validates Realtek-signed blobs; unsigned refused. LOCKDOWN_INTEGRITY_MAX disables FW updates.
- **PHY register write allowlist via ethtool** — direct PHY register writes restricted to documented PHY register space; some chip-internal registers (vendor extensions) refused without CAP_SYS_RAWIO.
- **WOL config CAP_NET_ADMIN** — closes "WOL prevents shutdown" class.
- **LED config CAP_SYS_ADMIN** — closes covert-channel-via-LED.
- **MAC change CAP_NET_ADMIN** — standard.
- **AER recovery rate-limit** — 3 in 60s.
- **ASPM disable for known-buggy chips immutable** — chips with known ASPM-stability issues have ASPM force-disabled; cannot be re-enabled at runtime (would crash the link). Closes a "user re-enables ASPM, system hangs" footgun-as-attack class.
- **Multi-instance consumer NIC** — when motherboard has 2x r8169 (rare on consumer; common on prosumer/server boards), instances are independent; cross-instance access refused.
- **Magic packet allowlist for WOL** — when WOL is enabled, only magic-packet matching the assigned MAC accepted; closes a path where a remote attacker on the same broadcast domain could wake unrelated machines.
- **NVRAM / EFUSE writes refused** — most r8169 chips don't have a userspace-writable NVRAM; if a write op is attempted, refused outright. Closes a path where a tampered driver could brick the NIC by writing bad MAC into EFUSE.

Per-doc rationale: r8169 has the largest installed base of any Ethernet silicon family but the smallest per-driver attack surface among the NICs Rookery documents. Defense focus: per-chip-version detection robustness (wrong-version → wrong-PHY-sequence → bricked link is the worst-case bug class), firmware-blob signature (closes substitution attack), PHY-init data immutability (closes runtime-PHY-tampering class), ASPM force-disable for buggy chips (a "footgun-as-attack" class — system hangs when ASPM is wrongly enabled). The huge per-chip-version table is the primary maintenance burden + the primary bug-source historically; the Rust translation's typed ChipVersion enum + per-version trait makes the dispatch type-safe + auditable.

## Open Questions

- [ ] Q1: 60+ chip versions — Rookery maintains all? Drop ancient (pre-RTL8168) versions? Recommendation: maintain all — RTL8169 silicon ships in long-life embedded gear (industrial routers, set-top boxes); deprecation only with explicit vendor coordination.
- [ ] Q2: RTL8125 dedicated driver (`rtase`) — separate Tier-3 needed? Currently r8169 covers RTL8125 too. Recommendation: rtase is for the server-grade variant with different MMIO layout; cover briefly here, defer to its own iteration.
- [ ] Q3: USB Realtek dongles use a separate `r8152` driver (USB-attached). Separate Tier-3 needed? Recommendation: yes, USB-attached has different concerns.
- [ ] Q4: ASPM per-chip immutability — should there be ANY override mechanism? Recommendation: no — let users with stability issues file bugs against the per-chip quirk table.

## Verification

- **Kani SAFETY**: prove ChipVersion::detect cross-validation correctness. Prove PHY init dispatch is exhaustive.
- **TLA+**: model PM state machine (D0/D3hot/D3cold) with concurrent WOL events.
- **Verus**: functional spec of the per-chip-version PHY init — for any chip version, applies the matching data table exactly.
- **Kani+Verus**: invariant that PHY init data tables are accessed read-only.
- **Integration**: r8169 on every chip-version test silicon — auto-neg, iperf3 throughput, WOL cycle, ASPM testing for known-buggy chips, runtime PM cycle stress.
- **Fuzz**: ethtool ioctl fuzzer; PHY register write fuzzer (must refuse out-of-allowlist).
- **Penetration**: unprivileged WOL config — refused. Unprivileged PHY register write — refused. Spoofed chip-version reads (via simulated MMIO mock) — driver refuses to bind.
