## Summary

i40e is the Intel 40/25 Gigabit Ethernet driver — covers the XL710 (40G), XXV710 (25G), X710 (10G derivative), and X722 (10G LOM variant) silicon. Family code-name Fortville (XL710) and Fortpond (X722). Deployed at scale in datacenters where 10G ixgbe was outgrown but 100G Mellanox/Broadcom wasn't yet justified — typical span 2014-2020 server generations. Still ships in many Dell/HPE/Lenovo/Supermicro motherboards and PCIe NICs. Largest installed base of Intel "modern" silicon (Admin-Queue + VirtChnl SR-IOV) before ice (100G).

The driver is ~50000 lines / ~20 .c files. Largest is `i40e_main.c` at ~16500 lines. Distinguishing architectural features from older ixgbe:
- **Admin Queue (AdminQ)** — DMA-based request/response ring for FW interaction (replaces the mailbox-register approach of older silicon).
- **VSI (Virtual Station Interface)** — abstraction unit for PF/VF/MAC-VLAN. A VSI has its own queues + MAC + VLAN filters. Multi-VSI per PF.
- **HMC (Host Memory Cache)** — per-PF DMA-mapped table of per-VSI/per-queue contexts; FW reads these for state. Replaces silicon-resident state tables of older designs.
- **VirtChnl SR-IOV** — PF-VF communication via virtchnl ABI (shared with `iavf` VF driver). Standardized across i40e + ice + (future) Intel SR-IOV designs.
- **DDP (Dynamic Device Personalization)** — parser plugin loading. PF can load custom parser profiles (e.g. for GTP, MPLS variants, custom tunneling) into the silicon at runtime.
- **Client interface** — `i40e_client.c` lets a sibling driver (the i40iw RDMA driver) attach for shared MSI-X + queue resources.

This Tier-3 covers ~50K lines.

## Rust translation posture

i40e is structurally similar to ice (newer Intel 100G) since both use AdminQ + VSI + VirtChnl. Translation:

- **`i40e-core` crate** — PCI lifecycle, netdev, NAPI, VSI mgmt, RX/TX descriptor handling, watchdog/reset.
- **AdminQ transport** typed: each opcode a method on `AdminQ`. Shared crate with ice (`intel-adminq`) if cross-driver reuse warranted.
- **VSI abstraction.** `Vsi<Kind>` with `Kind` in `{Main, MacVlan, Vf, Vmdq, Fdir}`. Different operations valid per VSI kind.
- **HMC mgmt.** Typed `Hmc` owning per-VSI/per-queue context allocations.
- **VirtChnl PF side.** PF handles per-VF VirtChnl messages; typed dispatch. Shared crate with iavf VF-side (`virtchnl-msg`).
- **DDP loader.** Custom parser profile loader; signature check + sanity bounds on profile binary.
- **Client interface** typed: `I40eClient` trait implemented by sibling drivers.
- **I40eAdapter<Phase>** typed lifecycle.

Grsec is mandatory. i40e has had CVEs (multiple SR-IOV VirtChnl bugs, AdminQ parsing bugs).

## Compatibility contract

REQ-1: PCI ID table — XL710, X710, XXV710, X722 variants. Frozen.

REQ-2: AdminQ ABI — fixed-size SQ + ARQ (Async Receive Queue) DMA-mapped. Cmd descriptors carry opcode + flags + DMA-buffer ptrs. Per-opcode response in CQ; async events on ARQ.

REQ-3: AdminQ opcode space — defined in `i40e_adminq_cmd.h`; hundreds of opcodes covering MAC/VSI/VLAN/RSS/PHY/DCB/NVM/PTP/manage.

REQ-4: VSI types — Main VSI, MacVlan VSI, VF VSI, VMDq VSI, FdirVSI. PF creates VSIs via `ADD_VSI` AdminQ cmd.

REQ-5: HMC layout — per-PF DMA region with: per-queue TX ctx, per-queue RX ctx, per-VSI ctx, FW-resident profile parser data. Layout per FW spec.

REQ-6: VirtChnl ABI (`include/linux/avf/virtchnl.h`) — frozen. PF↔VF msg opcodes: VIRTCHNL_OP_VERSION, _RESET_VF, _ADD_VSI, _GET_VF_RESOURCES, _CONFIG_TX/RX_QUEUE, _ADD/_DEL_MAC, _ADD/_DEL_VLAN, _CONFIG_RSS_LUT/_KEY, _ENABLE/DISABLE_QUEUES, _REQUEST_QUEUES, etc.

REQ-7: DDP profile loading — userspace loads a profile binary via ethtool/devlink; driver issues `WRITE_PERSONALIZATION_PROFILE` AdminQ cmd; FW activates the parser. Profile signature checked by FW.

REQ-8: DCB CEE + IEEE — both modes supported.

REQ-9: PTP — PHC + per-packet timestamping.

REQ-10: AF_XDP zerocopy — RX into UMEM-backed pool; TX from UMEM.

REQ-11: XDP_DRV — native at NAPI poll path; XDP_REDIRECT.

REQ-12: RDMA client (i40iw) — sibling driver attaches via `i40e_client.c` for shared resources.

REQ-13: NVRAM update — via AdminQ; signed FW image.

REQ-14: Reset coordinator — multiple reset sources (FW timeout, PF reset request, AER); coordinated through a single reset path with rate-limit.

REQ-15: Self-diag — `i40e_diag.c` runs HW self-test (register accessibility, eeprom checksum, IRQ test) via ethtool selftest.

## Architecture

**AdminQ transport.** Driver↔FW communication via two DMA-mapped rings: SQ (driver-write, FW-read for commands) + ARQ (FW-write, driver-read for async events + cmd responses). Each cmd descriptor has 32B fixed header + optional indirect DMA buffer for cmd payload. Cmd timeout enforced.

**VSI abstraction.** A VSI is the unit of FW-side network function. Per VSI: own queues, own MAC filters, own VLAN filters, own RSS table, own scheduler element. Types:
- **Main VSI** — primary PF netdev.
- **MacVlan VSI** — per-MAC-VLAN secondary netdev under a PF.
- **VF VSI** — per-VF; the VF sees this as its primary NIC.
- **VMDq VSI** — VirtualMachineDeviceQueue, hypervisor-controlled but VM-bound.
- **FdirVSI** — Flow Director match-and-action.

**HMC (Host Memory Cache).** Per-PF DMA region with per-queue + per-VSI contexts. FW reads from HMC for queue/VSI state (saves silicon SRAM). Layout strict per FW; driver writes contexts at queue/VSI create time.

**VirtChnl SR-IOV.** Standardized ABI between PF (driver) and VF (iavf driver, runs in guest). VF cannot directly do anything privileged — VF sends VirtChnl messages to PF, PF validates + executes. Standard messages: get caps, configure queues, add MAC, add VLAN, RSS config, enable/disable queues. PF side dispatches via opcode table.

**DDP (Dynamic Device Personalization).** Allows loading custom parser profiles into FW at runtime. Profile is a binary blob: parser state machine + lookup tables. Used for protocols not supported by base FW (GTP variants, MPLS-over-UDP, custom encapsulation). Profile signature checked.

**Client interface.** `i40e_client.c` defines a callback API. Sibling drivers (currently `i40iw` for RDMA) register; PF allocates shared queues + MSI-X vectors; sibling drives the queues for its protocol.

**Reset coordination.** Multiple reset sources: FW heartbeat miss, AER, PF reset request. All funnel through `i40e_handle_reset_warning` which serializes + rate-limits.

## Hardening

- AdminQ cmd timeout enforced.
- AdminQ response length validated per-opcode.
- VirtChnl msg validated against opcode-specific schema.
- DDP profile signature checked + size bounded.
- HMC ctx writes bounded by HMC layout.
- VSI count capped per-PF.
- Reset rate-limited.
- Client API: sibling-driver registration validated (one client at a time per type).

## Grsecurity/PaX-style Reinforcement

This driver inherits the standard PaX/Grsecurity surface and reinforces it with:

- **PAX_USERCOPY** — ethtool / devlink / DDP load ingress per-opcode size validation.
- **PAX_KERNEXEC** — netdev_ops, ethtool_ops, AdminQ opcode dispatch, VirtChnl PF dispatcher, DCB ops, PTP ops, XDP ops, client API callbacks, AER handlers all `const` after init in W^X memory.
- **PAX_RANDKSTACK** — ethtool / netlink / sysfs / devlink entry paths per-syscall stack-rerandomized.
- **PAX_REFCOUNT** — `i40e_pf` refcount, per-VSI refcount, per-ring refcount, per-VF refcount, client refcount, AdminQ in-flight cmd refcount all saturating.
- **PAX_MEMORY_SANITIZE** — AdminQ cmd buffers cleared after send. NVRAM-update payload cleared (signed FW image contents). DDP profile load buffer cleared after submission. Per-VF context cleared on VF disable.
- **PAX_UDEREF** — SMAP/SMEP on every user-facing entry.
- **PAX_RAP / kCFI** — netdev_ops, AdminQ dispatch, VirtChnl dispatcher, DCB ops, client API callbacks all kCFI-signed.
- **GRKERNSEC_HIDESYM** — debugfs scrubbed; reveals AdminQ addresses, VSI seid mappings, HMC region pointers.
- **GRKERNSEC_DMESG** — i40e verbose logs (VirtChnl trace revealing per-VF state, DDP profile load details) restricted from non-root.
- **CAP_NET_ADMIN required** — every state-changing op.
- **VirtChnl PF dispatcher closed-allowlist** — only known opcodes for the negotiated VirtChnl version accepted; unknown opcodes drop with rate-limited warn. Closes "compromised VF sends new opcode" class.
- **VirtChnl msg length validated per-opcode** — closes parser-OOB class.
- **VF VirtChnl resource caps strict** — VF cannot request more queues/MSI-X/RSS-slots than its assigned quota.
- **VF spoof-check HW-enforced** — same as ixgbe.
- **DDP profile signature mandatory** — Intel-signed profiles only; LOCKDOWN bypass-disabled.
- **DDP profile size capped** — closes "huge profile OOMs kernel" class.
- **NVRAM update via FW signature** — Intel-signed images; LOCKDOWN_INTEGRITY_MAX disables NVM updates.
- **Client API per-protocol single-instance** — only one RDMA client per PF; closes "rogue out-of-tree kmod registers as client" attack.
- **Reset rate-limit** — 3 in 60s; closes "AER storm" + "FW heartbeat gaslight" classes.
- **AdminQ cmd timeout strict** — bounded wait; past timeout, reset triggered.
- **HMC ctx write validation** — ctx fields validated against expected ranges before HMC write; closes "writing garbage ctx → FW does undefined things" class.
- **AF_XDP UMEM access mmu-notifier-strict** — same as ixgbe.
- **PTP CAP_SYS_TIME** — not CAP_NET_ADMIN.
- **Self-diag CAP_SYS_ADMIN** — `i40e_diag_reg_test` is intrusive; gated.
- **Debugfs CAP_SYS_ADMIN** — writes; reads scrub addresses.

Per-doc rationale: i40e is the bridge from older ixgbe-era SR-IOV (mailbox-register) to the modern Intel admin-queue + VirtChnl SR-IOV pattern used by ice and forward. The VirtChnl ABI is shared with iavf (VF driver) and (modulo extensions) with ice. The Rust translation reuses the VirtChnl typed-message + AdminQ transport across i40e + ice + (future) iavf. Historical CVE concentration in VirtChnl PF dispatcher (closed by opcode allowlist + per-opcode size). DDP profile loading adds a new attack surface (custom parser → wrong-parse exfil) — closed by Intel-signed-profile-only + size cap.

## Verification

- **Kani SAFETY**: prove VirtChnl PF dispatcher allowlist is exhaustive; AdminQ cmd timeout enforced (no infinite wait).
- **TLA+**: model VirtChnl PF/VF concurrent paths during VF reset; check no orphaned PF state after VF reset.
- **Verus**: functional spec of DDP profile parser — for valid signed profile, loads into HMC; for invalid signature/oversized, returns -EINVAL.
- **Kani+Verus**: invariant that every created VSI is tracked + released at PF teardown; no leak.
- **Integration**: XL710 + XXV710 stress — 40G + 25G iperf3, SR-IOV 32-VF stress, DDP profile load+swap, AF_XDP zerocopy, AER recovery, NVRAM-flash. PTP sync. i40iw RDMA over PFC fabric.
- **Fuzz**: VirtChnl msg fuzzer; AdminQ response fuzzer; DDP profile fuzzer.
- **Penetration**: compromised VF attempting PF-scope VirtChnl cmd — refused. Unsigned DDP profile — refused.
