## Summary

The cfg80211 **core** owns the per-`wiphy` and per-`wireless_dev` (wdev) lifecycle that every Wi-Fi driver in the Linux tree plugs into. A driver allocates state by calling `wiphy_new_nm(ops, sizeof_priv, requested_name)` which returns a `struct wiphy` embedded inside a `struct cfg80211_registered_device` (rdev); the driver then fills in supported bands, channels, interface modes, NL80211 features, cipher suites, regulatory hints, and `cfg80211_ops` callbacks (~200 function pointers covering scan/connect/AP/mesh/IBSS/P2P/NAN/PMSR/TDLS/...); finally the driver calls `wiphy_register(wiphy)`. Core validates the driver's iface-combinations matrix, sanity-checks per-band channel tables, registers the rdev in `cfg80211_rdev_list` under RTNL, creates `/sys/class/ieee80211/phy<idx>/`, creates `/sys/kernel/debug/ieee80211/phy<idx>/`, registers an rfkill node, applies the regulatory hint (`regulatory_hint_user`/`regulatory_hint`/cell-base/CRDA fall-back), and broadcasts `NL80211_CMD_NEW_WIPHY` to userspace listeners (wpa_supplicant, iwd, hostapd, iw, NetworkManager). Per-`wireless_dev`: a driver creates a netdev whose `dev->ieee80211_ptr = wdev`; the cfg80211 netdev notifier (`cfg80211_netdev_notifier_call`) hooks into the generic netdev notifier chain and transitions per `NETDEV_POST_INIT` → wdev init → `NETDEV_REGISTER` → wdev list-add → `NETDEV_UP` → opencount++ + `cfg80211_update_iface_num` → `NETDEV_GOING_DOWN` → leave + cancel work → `NETDEV_DOWN` → opencount-- + abort scan → `NETDEV_UNREGISTER` → `_cfg80211_unregister_wdev`. Per-NL80211: an SO_RAW NETLINK_GENERIC family `nl80211` ships ~200 commands (`NL80211_CMD_*`) and ~400 attribute types; `nl80211.c` owns the dispatch table — each command resolves to a `cfg80211_ops` callback via `rdev_<op>(rdev, ...)` shim (e.g., `NL80211_CMD_TRIGGER_SCAN` → `rdev->ops->scan(...)`). Per-scan: `cfg80211_scan_done` schedules `__cfg80211_scan_done` on the wiphy-work queue; per-connect: `cfg80211_conn_work` drives the SME state-machine (auth → assoc → connect); per-disconnect: `cfg80211_autodisconnect_wk`; per-station: `cfg80211_new_sta`/`cfg80211_del_sta` events bubble up to userspace; per-regulatory: `regulatory_init` loads the built-in `cfg80211_regdomain` + signature-verified CRDA payload, applies country-code updates either from the driver (`REGULATORY_CUSTOM_REG`), the user (CRDA `iw reg set`), the 11d/11h beacon (`regulatory_hint_country_ie`), or self-managed devices (`REGULATORY_WIPHY_SELF_MANAGED`); per-channel allocation: `cfg80211_chandef_*` enforces channel-width/center-freq/punctured-bitmap correctness against the wiphy's bands. Per-`ieee80211_pcb` host-driver integration: cfg80211 exposes ops slot (`cfg80211_ops`) that mac80211 fills for SoftMAC drivers; FullMAC drivers fill cfg80211_ops directly. Critical for: every Wi-Fi driver in `drivers/net/wireless/`, every userspace tool (iw, iwd, wpa_supplicant, hostapd, NetworkManager, iwconfig legacy).

This Tier-3 covers `net/wireless/core.c` (~2039 lines) and the directly load-bearing helpers it dispatches into for cross-file invariants (`nl80211.c` command-table contract, `reg.c` regulatory integration, the per-netdev notifier callbacks).

## Compatibility contract

REQ-1: `struct wiphy` (UAPI-visible fields; ABI-stable for in-tree drivers + nl80211 listeners):
- `perm_addr[6]`: per-permanent MAC; if `addresses` is provided it equals `addresses[0]`.
- `addresses, n_addresses`: per-virtual-iface MAC pool.
- `bands[NUM_NL80211_BANDS]`: per-band `ieee80211_supported_band*` (2.4 GHz, 5 GHz, 6 GHz, 60 GHz, S1G, LC).
- `interface_modes`: per-supported iftype bitmask (STA/AP/IBSS/MESH/MONITOR/P2P_CLIENT/P2P_GO/P2P_DEVICE/OCB/NAN/AP_VLAN/WDS-deprecated).
- `iface_combinations, n_iface_combinations`: per-`ieee80211_iface_combination[]` describing concurrent operation matrices (max channels, allowed iftype combos, radar detect widths, BI minima, …).
- `radio, n_radio, radio_cfg`: per-MLO radio descriptors (Wi-Fi 7 multi-link operation).
- `flags`: per-WIPHY_FLAG_* bitmask — `WIPHY_FLAG_NETNS_OK`, `WIPHY_FLAG_PS_ON_BY_DEFAULT`, `WIPHY_FLAG_HAVE_AP_SME`, `WIPHY_FLAG_SUPPORTS_FW_ROAM`, `WIPHY_FLAG_REPORTS_OBSS`, `WIPHY_FLAG_OFFCHAN_TX`, …
- `features`: per-NL80211_FEATURE_* bitmask.
- `ext_features[]`: per-NL80211_EXT_FEATURE_* bitmap.
- `regulatory_flags`: per-REGULATORY_* bitmask (CUSTOM_REG, STRICT_REG, COUNTRY_IE_FOLLOW_POWER, COUNTRY_IE_IGNORE, WIPHY_SELF_MANAGED, DISABLE_BEACON_HINTS, ENABLE_RELAX_NO_IR, IGNORE_REG_UPDATE).
- `regd`: per-RCU-protected `ieee80211_regdomain` (NULL until first hint applied).
- `cipher_suites, n_cipher_suites`: per-driver-supported cipher suites (WEP/TKIP/CCMP/GCMP/AES-CMAC/AES-GMAC).
- `wowlan`, `wowlan_config`: per-WoWLAN trigger surface + active config.
- `coalesce`: per-CoalEsce rule descriptor.
- `pmsr_capa`: per-PMSR / FTM (802.11mc) capability.
- `nan_supported_bands`, `nan_capa`: per-NAN capability.
- `retry_short`, `retry_long`: per-MIB defaults (7, 4).
- `frag_threshold`, `rts_threshold`: per-MIB defaults (-1, -1 = disabled).
- `coverage_class`, `max_num_csa_counters`: per-MIB extras.
- `max_sched_scan_plans`, `max_sched_scan_plan_interval`: per-sched-scan limits.
- `bss_select_support`: per-BSS-selection-rule bitmask.
- `vendor_commands, n_vendor_commands`, `vendor_events, n_vendor_events`: per-vendor extensions surface.
- `rfkill`: per-rfkill instance.
- `mtx`: per-wiphy mutex (the "wiphy lock") — held by every nl80211 command + most internal mutations.
- `wdev_list`: per-vif list under wiphy_mtx.
- `dev`: embedded `struct device` for sysfs.
- `debugfsdir`: per-wiphy debugfs root.

REQ-2: `struct cfg80211_registered_device` (rdev) — kernel-internal, follows `wiphy` in the same allocation:
- `wiphy`: per-public state.
- `ops`: per-`cfg80211_ops` vtable.
- `wiphy_idx`: per-monotonic counter (assigned from `wiphy_counter` `atomic_t`).
- `list`: per-`cfg80211_rdev_list` linkage (RCU).
- `bss_list, bss_lock`, `bss_entries_count`, `bss_generation`: per-BSS database for scan results.
- `scan_req`, `int_scan_req`, `scan_msg`, `sched_scan_req_list`: per-scan state.
- `mlme_unreg_lock`, `beacon_registrations`, `mgmt_registrations_update_wk`: per-mgmt-frame registration.
- `event_work, event_list`, `event_lock`: per-pending-event queue.
- `wiphy_work, wiphy_work_list, wiphy_work_lock`: per-wiphy_work queue (drain-bounded at 100).
- `destroy_work, destroy_iface_wk`: per-deferred-vif-destroy.
- `propagate_radar_detect_wk, propagate_cac_done_wk`: per-DFS work.
- `background_cac_abort_wk, background_cac_done_wk`: per-background-CAC (802.11h channel-availability check).
- `dfs_update_channels_wk`: per-DFS channel update delayed work.
- `scan_done_wk`, `sched_scan_stop_wk`, `sched_scan_res_wk`: per-scan-completion work.
- `conn_work`: per-SME (Station Mode connect state-machine) work — runs `cfg80211_conn_work`.
- `rfkill_block, rfkill_ops`: per-rfkill block work + ops.
- `dev_wait, opencount`: per-up-iface counter + wait queue (`wiphy_unregister` blocks until opencount==0).
- `devlist_generation, wdev_id`: per-list-generation counter + per-wdev-identifier allocator.
- `coalesce`: per-coalesce-config copy.
- `wowlan_config`: per-driver-set WoWLAN config (under rdev_set_wakeup).
- `regd`: shadow of `wiphy.regd` accessed under RTNL.

REQ-3: `struct wireless_dev` (wdev) — per-vif (one per netdev for STA/AP/MESH/IBSS/MON/P2P_GO/P2P_CLIENT; one without netdev for P2P_DEVICE + NAN):
- `wiphy`: per-back-pointer.
- `iftype`: per-NL80211_IFTYPE_*.
- `netdev`: per-netdev pointer (NULL for P2P_DEVICE / NAN).
- `address[6]`: per-vif MAC.
- `identifier`: per-`++rdev->wdev_id` allocated identifier (0 reserved; assigned at first register).
- `registered, registering`: per-state-flags.
- `is_running`: per-up-flag for non-netdev wdev.
- `ps`, `ps_timeout`: per-power-save state.
- `event_list, event_lock`: per-pending-event queue.
- `mgmt_registrations, mgmt_registrations_update_wk`: per-mgmt-frame registration.
- `pmsr_list, pmsr_lock, pmsr_free_wk`: per-PMSR/FTM measurement state.
- `cqm_rssi_work`: per-CQM-RSSI threshold wiphy_work.
- `disconnect_wk, autodisconnect_wk`: per-deferred-disconnect.
- `links[IEEE80211_MLD_MAX_NUM_LINKS]`: per-MLO link state (Wi-Fi 7).
- `radio_mask`: per-MLO radio occupancy mask.
- `u`: per-iftype union (`u.ibss`, `u.mesh`, `u.client`, `u.ap`, `u.ocb`, `u.nan`, `u.mon`).
- `wext`: per-legacy wext state (CONFIG_CFG80211_WEXT).
- `connect_keys`: per-pre-connect key cache.
- `conn`: per-SME-state-machine context (assoc/auth/connect).
- `nl_owner_dead`: per-NL80211 owner-process exit flag.
- `use_4addr`: per-WDS (4-address mode) flag.
- `valid_links, active_links`: per-MLO bitmasks.

REQ-4: `wiphy_new_nm(ops, sizeof_priv, requested_name) -> *wiphy`:
- Sanity-WARN on missing ops-pair invariants: `add_key && (!del_key || !set_default_key)`; `auth && (!assoc || !deauth || !disassoc)`; `connect && !disconnect`; `join_ibss && !leave_ibss`; `add_virtual_intf && !del_virtual_intf`; `add_station && !del_station`; `add_mpath && !del_mpath`; `join_mesh && !leave_mesh`; `start_p2p_device && !stop_p2p_device`; `start_ap && !stop_ap`; `join_ocb && !leave_ocb`; `suspend && !resume`; `sched_scan_start && !sched_scan_stop`; `remain_on_channel && !cancel_remain_on_channel`; `tdls_channel_switch && !tdls_cancel_channel_switch`; `add_tx_ts && !del_tx_ts`.
- Allocate `sizeof(rdev) + sizeof_priv`; zeroed via kzalloc; on alloc fail return NULL.
- `rdev->ops = ops`.
- `rdev->wiphy_idx = atomic_inc_return(&wiphy_counter) - 1` (wraps-detected; on negative atomic-dec + free + NULL).
- `dev_set_name(&rdev->wiphy.dev, requested_name ?: "phy%d", wiphy_idx)`; under RTNL if requested_name supplied (so `cfg80211_dev_check_name` can validate uniqueness).
- `mutex_init(&wiphy.mtx); INIT_LIST_HEAD(&wiphy.wdev_list); INIT_LIST_HEAD(&beacon_registrations); spin_lock_init(&beacon_registrations_lock); spin_lock_init(&bss_lock); INIT_LIST_HEAD(&bss_list); INIT_LIST_HEAD(&sched_scan_req_list)`.
- `wiphy_work_init(&scan_done_wk, __cfg80211_scan_done); INIT_DELAYED_WORK(&dfs_update_channels_wk, cfg80211_dfs_channels_update_work)`.
- If CONFIG_CFG80211_WEXT: `wiphy.wext = &cfg80211_wext_handler`.
- `device_initialize(&wiphy.dev); wiphy.dev.class = &ieee80211_class; wiphy.dev.platform_data = rdev; device_enable_async_suspend(&wiphy.dev)`.
- `INIT_WORK(destroy_work, cfg80211_destroy_iface_wk); wiphy_work_init(sched_scan_stop_wk, cfg80211_sched_scan_stop_wk); INIT_WORK(sched_scan_res_wk, cfg80211_sched_scan_results_wk); INIT_WORK(propagate_radar_detect_wk, ...); INIT_WORK(propagate_cac_done_wk, ...); INIT_WORK(mgmt_registrations_update_wk, ...); INIT_WORK(wiphy_work, cfg80211_wiphy_work); INIT_LIST_HEAD(&wiphy_work_list); spin_lock_init(&wiphy_work_lock)`.
- If CONFIG_CFG80211_DEFAULT_PS: `wiphy.flags |= WIPHY_FLAG_PS_ON_BY_DEFAULT`.
- `wiphy_net_set(&wiphy, &init_net)`.
- `rfkill_ops.set_block = cfg80211_rfkill_set_block; wiphy.rfkill = rfkill_alloc(dev_name, &wiphy.dev, RFKILL_TYPE_WLAN, &rfkill_ops, rdev)`; on alloc-fail `wiphy_free` + NULL.
- `INIT_WORK(rfkill_block, cfg80211_rfkill_block_work); INIT_WORK(conn_work, cfg80211_conn_work); INIT_WORK(event_work, cfg80211_event_work); INIT_WORK(background_cac_abort_wk, ...); INIT_DELAYED_WORK(background_cac_done_wk, ...); init_waitqueue_head(&dev_wait)`.
- Initialize MIB defaults: `retry_short = 7; retry_long = 4; frag_threshold = -1; rts_threshold = -1; coverage_class = 0; max_num_csa_counters = 1; max_sched_scan_plans = 1; max_sched_scan_plan_interval = U32_MAX`.
- Return `&rdev->wiphy`.

REQ-5: `wiphy_register(wiphy) -> int`:
- Validate WoWLAN consistency under CONFIG_PM.
- Validate `NL80211_FEATURE_TDLS_CHANNEL_SWITCH` consistency with `tdls_channel_switch` op.
- Validate NAN: needs `start_nan`, `stop_nan`, `add_nan_func`, `del_nan_func`, supports `NL80211_BAND_2GHZ`.
- Validate NAN_DATA: requires `nan_capa.phy.ht.ht_supported`.
- Reject `BIT(NL80211_IFTYPE_WDS)` (deprecated).
- Validate `pmsr_capa` consistency (asap or non-asap, preambles/bandwidths bitmasks within enumerated sets).
- Validate regulatory flags: `REGULATORY_WIPHY_SELF_MANAGED` mutually exclusive with `REGULATORY_CUSTOM_REG | REGULATORY_STRICT_REG | REGULATORY_COUNTRY_IE_FOLLOW_POWER | REGULATORY_COUNTRY_IE_IGNORE`.
- Validate coalesce/AP-SME-cap/MAC-ACL/BSS-select/4WAY-HSHK consistency.
- Copy `addresses[0]` to `perm_addr` if `addresses` is set.
- Validate ifmodes (≥1, no invalid bit 0).
- `res = wiphy_verify_combinations(wiphy)`.
- For each band: sanity-check `n_channels`, `n_bitrates` (≤32, 0 only on 60GHz/S1GHZ), 6 GHz forbids legacy HT/VHT (HE+); initialize `channels[i].orig_flags / orig_mag / orig_mpwr / band`.
- For each per-band iftype-data block: validate HE/HT/VHT/EHT capabilities, AP vs non-AP separation, MLO link counts.
- `rdev->wiphy.registered = true` post-registration.
- `wiphy_regulatory_register(wiphy)` (applies REGULATORY_*; sends signal to reg.c).
- `device_add(&wiphy.dev)` → creates `/sys/class/ieee80211/phy<idx>/`.
- `debugfsdir = debugfs_create_dir(name, ieee80211_debugfs_dir)` → `/sys/kernel/debug/ieee80211/phy<idx>/`.
- `rfkill_register(wiphy.rfkill)`; if blocked at register, propagate via `cfg80211_rfkill_set_block(true)`.
- Under RTNL: `list_add_rcu(&rdev->list, &cfg80211_rdev_list); cfg80211_rdev_list_generation++`.
- `nl80211_notify_wiphy(rdev, NL80211_CMD_NEW_WIPHY)`.
- Return 0.

REQ-6: `wiphy_unregister(wiphy)`:
- `wait_event(rdev->dev_wait, opencount == 0)` — blocks until every wdev down.
- `rfkill_unregister(rfkill)`.
- Under RTNL + `wiphy_lock`: `nl80211_notify_wiphy(rdev, NL80211_CMD_DEL_WIPHY)`.
- `rdev->wiphy.registered = false`.
- Debug-assert `wdev_list` empty.
- `debugfs_remove_recursive(debugfsdir)`.
- `list_del_rcu(&rdev->list); synchronize_rcu()`.
- `wiphy_regulatory_deregister(wiphy)`.
- `cfg80211_rdev_list_generation++`.
- `device_del(&wiphy.dev)`.
- CONFIG_PM: if `wowlan_config && set_wakeup`: `rdev_set_wakeup(rdev, false)`.
- `cfg80211_process_wiphy_works(rdev, NULL)` — drain residual wiphy_work.
- `cancel_work_sync` on `wiphy_work, rfkill_block, conn_work`.
- `flush_work` on `event_work, destroy_work, propagate_radar_detect_wk, propagate_cac_done_wk, mgmt_registrations_update_wk, background_cac_abort_wk`.
- `cancel_delayed_work_sync` on `dfs_update_channels_wk, background_cac_done_wk`.
- `cfg80211_rdev_free_wowlan(rdev); cfg80211_free_coalesce(coalesce)`.

REQ-7: `wiphy_free(wiphy)`:
- `kfree(wiphy->radio_cfg); put_device(&wiphy->dev)` — final reference drop triggers `cfg80211_dev_free` via the class release.

REQ-8: `cfg80211_dev_free(rdev)` (class release callback):
- Drain residual `wiphy_work` (asserted empty); `cancel_work_sync(&wiphy_work)`.
- `rfkill_destroy(rfkill)`.
- Free `beacon_registrations`, `bss_list` (each via `cfg80211_put_bss`).
- `mutex_destroy(&wiphy.mtx)`.
- Free `wiphy.regd` if non-NULL (only set on failure-paths that never reached `wiphy_register`).
- `kfree(rdev)`.

REQ-9: `cfg80211_netdev_notifier_call(nb, state, ptr) -> int`:
- Resolve `dev = netdev_notifier_info_to_dev(ptr); wdev = dev->ieee80211_ptr`.
- If `!wdev`: return NOTIFY_DONE (non-wireless netdev).
- Resolve `rdev = wiphy_to_rdev(wdev->wiphy)`.
- `WARN_ON(wdev->iftype == NL80211_IFTYPE_UNSPECIFIED)`.
- Switch state:
  - `NETDEV_POST_INIT`: `SET_NETDEV_DEVTYPE(dev, &wiphy_type); wdev->netdev = dev; dev->netns_immutable = true; cfg80211_init_wdev(wdev)`.
  - `NETDEV_REGISTER`: if `!wdev->registered`: `guard(wiphy)(&rdev->wiphy); cfg80211_register_wdev(rdev, wdev)`.
  - `NETDEV_UNREGISTER`: if registered && !registering: `guard(wiphy)(&rdev->wiphy); _cfg80211_unregister_wdev(wdev, false)`.
  - `NETDEV_GOING_DOWN`: `cfg80211_leave(rdev, wdev, -1); scoped_guard(wiphy, &rdev->wiphy) cfg80211_remove_links(wdev); cancel_work_sync(disconnect_wk); cancel_work_sync(pmsr_free_wk)`.
  - `NETDEV_DOWN`: `wiphy_lock; cfg80211_update_iface_num(rdev, iftype, -1)`; if scan_req references this wdev, mark aborted + `___cfg80211_scan_done(rdev, false)`; for each sched-scan: if owner == wdev, stop; `opencount--; wiphy_unlock; wake_up(dev_wait)`.
  - `NETDEV_UP`: `wiphy_lock; cfg80211_update_iface_num(rdev, iftype, 1)`; per-iftype WEXT-mode handlers (IBSS join, STA connect, MESH join); `opencount++`; configure PS via `rdev_set_power_mgmt`; `wiphy_unlock`.
  - `NETDEV_PRE_UP`: `cfg80211_iftype_allowed` check → `-EOPNOTSUPP`; `rfkill_blocked` → `-ERFKILL`; NAN_DATA requires running NAN.
  - default: NOTIFY_DONE.
- `wireless_nlevent_flush(); return NOTIFY_OK`.

REQ-10: `cfg80211_init_wdev(wdev)`:
- `INIT_LIST_HEAD(event_list); spin_lock_init(event_lock); INIT_LIST_HEAD(mgmt_registrations); INIT_LIST_HEAD(pmsr_list); spin_lock_init(pmsr_lock); INIT_WORK(pmsr_free_wk, cfg80211_pmsr_free_wk)`.
- CONFIG_CFG80211_WEXT: `wext.default_key=-1; wext.default_mgmt_key=-1; wext.connect.auth_type=NL80211_AUTHTYPE_AUTOMATIC`.
- `wiphy_work_init(cqm_rssi_work, cfg80211_cqm_rssi_notify_work)`.
- `ps = (wiphy.flags & WIPHY_FLAG_PS_ON_BY_DEFAULT) ? true : false; ps_timeout = -1`.
- `radio_mask = BIT(wiphy.n_radio) - 1`.
- If iftype ∈ {STA, P2P_CLIENT, ADHOC} && !use_4addr: `netdev->priv_flags |= IFF_DONT_BRIDGE`.
- `INIT_WORK(disconnect_wk, cfg80211_autodisconnect_wk)`.

REQ-11: `cfg80211_register_wdev(rdev, wdev)` (ASSERT_RTNL + wiphy.mtx held):
- If `wdev->identifier == 0`: `wdev->identifier = ++rdev->wdev_id`.
- `list_add_rcu(&wdev->list, &rdev->wiphy.wdev_list); rdev->devlist_generation++; wdev->registered = true`.
- If `wdev->netdev`: `sysfs_create_link(&netdev->dev.kobj, &wiphy.dev.kobj, "phy80211")`.
- `nl80211_notify_iface(rdev, wdev, NL80211_CMD_NEW_INTERFACE)`.

REQ-12: `_cfg80211_unregister_wdev(wdev, unregister_netdev)`:
- `nl80211_notify_iface(rdev, wdev, NL80211_CMD_DEL_INTERFACE)`.
- `wdev->registered = false`.
- If `netdev`: `sysfs_remove_link(phy80211)`; if `unregister_netdev`: `unregister_netdevice(netdev)`.
- `list_del_rcu(&wdev->list); synchronize_net(); rdev->devlist_generation++`.
- `cfg80211_mlme_purge_registrations(wdev)`.
- Per-iftype cleanup:
  - P2P_DEVICE: `cfg80211_stop_p2p_device(rdev, wdev)`.
  - NAN: `cfg80211_stop_nan(rdev, wdev)`.

REQ-13: `cfg80211_register_netdevice(dev) -> int`:
- ASSERT_RTNL.
- Resolve `wdev = dev->ieee80211_ptr; rdev = wiphy_to_rdev(wdev->wiphy)`.
- `lockdep_assert_held(&rdev->wiphy.mtx)`.
- Set `wdev->registered = true; wdev->registering = true`.
- `register_netdevice(dev)` (triggers the notifier chain — but `NETDEV_REGISTER` notices `wdev->registered` already true and skips re-register).
- On success call `cfg80211_register_wdev(rdev, wdev)`.
- Clear `registering`; if failed, also clear `registered`.

REQ-14: `cfg80211_switch_netns(rdev, net) -> int`:
- Pre-cond: `wiphy.flags & WIPHY_FLAG_NETNS_OK` else `-EOPNOTSUPP`.
- For each wdev with a netdev: `netdev->netns_immutable = false; dev_change_net_namespace(netdev, net, "wlan%d"); netdev->netns_immutable = true`. On failure: revert in reverse order.
- Under `wiphy.mtx`: for each wdev with a netdev send `NL80211_CMD_DEL_INTERFACE` to old netns, then `NL80211_CMD_DEL_WIPHY`, swap netns via `wiphy_net_set`, `device_rename` (re-emits class-uevent on the new netns), `NL80211_CMD_NEW_WIPHY`, then `NL80211_CMD_NEW_INTERFACE` for each wdev.

REQ-15: rfkill integration:
- `cfg80211_rfkill_set_block(data, blocked)`: if blocked, under RTNL call `cfg80211_shutdown_all_interfaces(&rdev->wiphy)`.
- `cfg80211_shutdown_all_interfaces(wiphy)`: ASSERT_RTNL; first `dev_close(wdev->netdev)` for every wdev with a netdev; then under `wiphy_lock`, stop P2P_DEVICE and NAN (no netdev).
- `cfg80211_rfkill_poll(rfkill, data)`: per-driver poll under `wiphy_lock` via `rdev_rfkill_poll`.
- `cfg80211_rfkill_block_work(work)`: invoked by HW rfkill state change; calls `cfg80211_rfkill_set_block(rdev, true)` outside the IRQ context.
- `wiphy_rfkill_set_hw_state_reason(wiphy, blocked, reason)`: thin driver-facing wrapper.

REQ-16: `wiphy_work` runner — bounded queue:
- `wiphy_work_queue(wiphy, work)`: under `wiphy_work_lock`, if not already on the list, `list_add_tail(&work->entry, &rdev->wiphy_work_list); queue_work(system_unbound_wq, &rdev->wiphy_work)` (trace + flush deferred).
- `cfg80211_wiphy_work(work)` (the rdev->wiphy_work itself): `guard(wiphy)(&rdev->wiphy); cfg80211_process_wiphy_works(rdev, NULL)`.
- `cfg80211_process_wiphy_works(rdev, end)`: `lockdep_assert_held(&rdev->wiphy.mtx)`. Loop draining `wiphy_work_list` under `wiphy_work_lock`; for each `wk`: `list_del_init`, drop spinlock, `trace_wiphy_work_run`, `wk->func(wiphy, wk)`, retake lock; stop when `wk == end` or after 100 iterations (WARN + reset list) to bound work-queue runaway.
- `wiphy_work_cancel`: spin-lock + `list_del_init` if entry queued.
- `wiphy_work_flush`: spin-lock + check `list_empty(&work->entry)`; if not, `cfg80211_process_wiphy_works(rdev, work)`.
- `wiphy_delayed_work_*`: timer-armed variant. `wiphy_delayed_work_timer` queues on timer expiry. `wiphy_delayed_work_pending` reports both timer-armed and queued states.
- `wiphy_hrtimer_work_*`: hrtimer-armed variant (HARD_IRQSAFE).

REQ-17: NL80211 dispatch root (cross-file invariant with `nl80211.c`):
- NL80211 family registered in `nl80211_init` (called from `cfg80211_init`).
- `genl_family nl80211_fam` with `cmds[]` array indexed by `NL80211_CMD_*`.
- Per-command policy: `.policy = nl80211_policy`, `.doit`/`.dumpit`/`.maxattr`, `.flags = GENL_ADMIN_PERM | GENL_UNS_ADMIN_PERM | …`, `.internal_flags = NL80211_FLAG_NEED_NETDEV | NL80211_FLAG_NEED_WIPHY | …`.
- Pre-dispatch wrapper resolves the target rdev/wdev, takes `wiphy.mtx`, performs PoLP checks (capability + namespace match), invokes the `.doit` handler, and on return notifies userspace listeners.
- Per-ops shim layer (`net/wireless/rdev-ops.h`): `rdev_<op>(rdev, ...) -> { trace_<op>; ret = rdev->ops-><op>(...); trace_<op>_return; ret }`.

REQ-18: regulatory domain (cross-file invariant with `reg.c`):
- `regulatory_init()` (called from `cfg80211_init`): load built-in `cfg80211_regdomain` (worldwide / world-roaming); set `reg_pending_alpha2 = ""`; init `reg_pending_beacons` workqueue; register the reg-hint Genetlink interface; on CONFIG_CFG80211_REQUIRE_SIGNED_REGDB, verify the loaded regdomain against the signed reg-db blob in `certs/`.
- Per `wiphy_regulatory_register(wiphy)` (called from `wiphy_register`):
  - If `REGULATORY_CUSTOM_REG`: copy driver's custom regdomain into `wiphy.regd` (RCU-published).
  - If `REGULATORY_STRICT_REG`: enforce strict mode (no relaxation).
  - If `REGULATORY_WIPHY_SELF_MANAGED`: skip global hint applications; driver alone owns the regdomain.
  - Else: queue a `reg_initiator = NL80211_REGDOM_SET_BY_DRIVER` hint via `regulatory_hint`.
- `regulatory_hint(wiphy, alpha2)`: driver-supplied 2-letter country.
- `regulatory_hint_user(alpha2, user_hint_type)`: from `iw reg set` (`NL80211_CMD_REQ_SET_REG`) or CRDA userspace.
- `regulatory_hint_country_ie(wiphy, band, country_ie, len)`: 11d beacon-IE-derived hint.
- `regulatory_hint_indoor(true/false)`: from user (`iw reg set indoor`).
- `regulatory_hint_disconnect()`: from rdev (last STA disconnect).
- `regulatory_set_wiphy_regd(wiphy, regd)`: self-managed update path.
- Country-code update applies channel-flag overlays: NO_IR (no initiating radiation), NO_OUTDOOR, DFS, NO_HT40, NO_80MHZ, NO_160MHZ, NO_320MHZ, AUTO_BW; updates max EIRP per-channel.

REQ-19: scan / connect / disconnect dispatch:
- Scan: NL80211_CMD_TRIGGER_SCAN → `rdev->ops->scan(wiphy, scan_request)` → driver fires; on done, driver calls `cfg80211_scan_done(scan_request, info)` → wiphy_work-queued `__cfg80211_scan_done` → `nl80211_send_scan_done` + BSS-list update.
- Sched-scan: NL80211_CMD_START_SCHED_SCAN → `rdev->ops->sched_scan_start`; results: `cfg80211_sched_scan_results` queues `sched_scan_res_wk`; stop: `cfg80211_sched_scan_stop_wk`.
- Connect (SoftMAC): NL80211_CMD_CONNECT → `cfg80211_connect` → schedules `conn_work` (`cfg80211_conn_work`) which runs the SME state machine (Auth → Assoc → Connected); on completion, `cfg80211_connect_done` notifies userspace.
- Connect (FullMAC): driver-driven; FullMAC drivers expose `rdev->ops->connect` directly.
- Disconnect: NL80211_CMD_DISCONNECT → `cfg80211_disconnect`; on netdev going down without explicit disconnect: `cfg80211_autodisconnect_wk`.
- Stop link / MLO leave: `cfg80211_stop_link(wiphy, wdev, link_id)`.

REQ-20: station management:
- `NL80211_CMD_NEW_STATION` / `_DEL_STATION` / `_SET_STATION` / `_GET_STATION` / `_DUMP_STATION` → `rdev->ops->{add,del,change,get,dump}_station`.
- Events: `cfg80211_new_sta`, `cfg80211_del_sta`, `cfg80211_conn_failed`, `cfg80211_mgmt_tx_status`, `cfg80211_rx_mgmt`, … push events via `nl80211_send_event`.

REQ-21: per-netns lifecycle:
- `cfg80211_pernet_exit(net)`: on net-namespace exit, walk every rdev whose `wiphy_net == net` and move it back to `init_net` via `cfg80211_switch_netns(rdev, &init_net)` (only valid for `WIPHY_FLAG_NETNS_OK` wiphys; others remain pinned to init_net).

REQ-22: module init / exit ordering (`cfg80211_init`):
- `register_pernet_device(&cfg80211_pernet_ops)` →
- `wiphy_sysfs_init()` (registers `&ieee80211_class` and `wiphy_type` device-type) →
- `register_netdevice_notifier(&cfg80211_netdev_notifier)` →
- `nl80211_init()` (registers NL80211 genl family + commands) →
- `ieee80211_debugfs_dir = debugfs_create_dir("ieee80211", NULL)` →
- `regulatory_init()` →
- `cfg80211_wq = alloc_ordered_workqueue("cfg80211", WQ_MEM_RECLAIM)`.
- Exit is reverse order.

REQ-23: ieee80211_class + sysfs surface:
- `&ieee80211_class` defines `name = "ieee80211"`, `dev_release = cfg80211_dev_free`, plus `dev_attrs` (`index`, `macaddress`, `address_mask`, `addresses`, `name`).
- Sysfs hierarchy:
  - `/sys/class/ieee80211/phy<idx>/` — wiphy class device.
  - `/sys/class/ieee80211/phy<idx>/index`, `macaddress`, `name`, `address_mask`, `addresses`, `uevent`, `power/`, `subsystem/`.
  - `/sys/class/ieee80211/phy<idx>/rfkill<K>/` symlink to rfkill node.
  - `/sys/class/net/wlanX/phy80211` symlink → `/sys/class/ieee80211/phy<idx>/` (created by `cfg80211_register_wdev`).
- Debugfs: `/sys/kernel/debug/ieee80211/phy<idx>/` per-wiphy.

REQ-24: `ieee80211_pcb` host-driver integration:
- SoftMAC drivers create their wiphy via `ieee80211_alloc_hw` (from `net/mac80211/main.c`) which internally calls `wiphy_new_nm` with the mac80211-provided `cfg80211_ops`; the driver's `ieee80211_ops` are layered atop the mac80211-provided ones.
- FullMAC drivers (e.g., `brcmfmac`, parts of `mt76`, `iwlmvm`-tower vendor) call `wiphy_new`/`wiphy_new_nm` directly with their own `cfg80211_ops`.
- `wiphy_priv(wiphy)` returns the per-driver `sizeof_priv`-byte private area placed immediately after `cfg80211_registered_device` in the same allocation.

## Acceptance Criteria

- [ ] AC-1: `wiphy_new_nm` with required-pair op missing (e.g., `connect && !disconnect`): WARN_ON fires; allocation still succeeds (warn-only).
- [ ] AC-2: `wiphy_register` with valid bands + ifmodes + iface_combinations: succeeds; `/sys/class/ieee80211/phy<idx>/` exists; `/sys/kernel/debug/ieee80211/phy<idx>/` exists; `cfg80211_rdev_list` contains the rdev; `NL80211_CMD_NEW_WIPHY` event delivered to genl listeners.
- [ ] AC-3: `wiphy_register` with `BIT(NL80211_IFTYPE_WDS)` in interface_modes: returns `-EINVAL` (WDS deprecated).
- [ ] AC-4: `wiphy_register` with conflicting regulatory flags (SELF_MANAGED + CUSTOM_REG): returns `-EINVAL`.
- [ ] AC-5: `wiphy_register` with `n_channels==0` on a populated band: returns `-EINVAL`.
- [ ] AC-6: `wiphy_register` with `band == NL80211_BAND_6GHZ && ht_supported`: returns `-EINVAL` (6 GHz forbids legacy HT/VHT).
- [ ] AC-7: `wiphy_register` with `n_bitrates > 32`: returns `-EINVAL`.
- [ ] AC-8: `wiphy_unregister` blocks until `opencount == 0`; after unblock, `cfg80211_rdev_list` no longer contains the rdev; `NL80211_CMD_DEL_WIPHY` event delivered; debugfsdir removed; rfkill destroyed.
- [ ] AC-9: `wiphy_unregister` with non-empty `wdev_list`: `WARN_ON` fires (caller bug).
- [ ] AC-10: `cfg80211_netdev_notifier_call(NETDEV_POST_INIT)` on a wdev-bearing netdev: `wdev->netdev` set; `netns_immutable=true`; `cfg80211_init_wdev` ran (`event_list` initialized, default PS configured per WIPHY flag).
- [ ] AC-11: `cfg80211_netdev_notifier_call(NETDEV_UP)`: under `wiphy_lock`, `iface_num[iftype]++`; `opencount++`; PS sent to driver.
- [ ] AC-12: `cfg80211_netdev_notifier_call(NETDEV_DOWN)`: `iface_num[iftype]--`; if active scan_req for this wdev, aborted + scan-done emitted; sched-scans torn down; `opencount--`; `wake_up(dev_wait)`.
- [ ] AC-13: `cfg80211_netdev_notifier_call(NETDEV_PRE_UP)` with rfkill blocked: returns `-ERFKILL`.
- [ ] AC-14: `cfg80211_netdev_notifier_call(NETDEV_PRE_UP)` for NAN_DATA without running NAN: returns `-ENOLINK`.
- [ ] AC-15: `cfg80211_rfkill_set_block(true)`: all wdev with netdev are `dev_close`'d under RTNL; all P2P_DEVICE and NAN wdev stopped under `wiphy_lock`.
- [ ] AC-16: `cfg80211_switch_netns` on `!WIPHY_FLAG_NETNS_OK`: returns `-EOPNOTSUPP`.
- [ ] AC-17: `cfg80211_switch_netns` partial failure mid-walk: reverts every already-moved wdev to source netns.
- [ ] AC-18: `cfg80211_process_wiphy_works` runaway: at >100 work items per drain, list is reset and `WARN_ON` fires (defense against work-storm).
- [ ] AC-19: `wiphy_work_queue` of an already-queued work: NOP (no double-enqueue).
- [ ] AC-20: NL80211_CMD_TRIGGER_SCAN dispatched: `rdev->ops->scan` invoked under `wiphy_lock`; on `cfg80211_scan_done`, `__cfg80211_scan_done` runs on the wiphy_work queue and `NL80211_CMD_NEW_SCAN_RESULTS` is emitted.
- [ ] AC-21: Driver-supplied regulatory_hint at register-time: applied via `wiphy_regulatory_register`; `wiphy->regd` becomes non-NULL post-registration.
- [ ] AC-22: `regulatory_hint_user("US", ...)` from CRDA: country code propagates to every non-self-managed wiphy; per-channel flags updated.

`Cfg80211::wiphy_new_nm(ops, sizeof_priv, requested_name) -> Result<*mut Wiphy, ()>`:
1. WARN_ON the documented `cfg80211_ops` pairing invariants (REQ-4).
2. `rdev = kzalloc(size_of::<Rdev>() + sizeof_priv, GFP_KERNEL)?`.
3. `rdev.ops = ops`.
4. `rdev.wiphy_idx = atomic_inc_return(&WIPHY_COUNTER) - 1`; if negative-wrap: `atomic_dec`, free, return None.
5. Name the wiphy:
   - If `requested_name`: under RTNL `cfg80211_dev_check_name`; on success `dev_set_name(... requested_name)`; on check failure fall through to "phyN".
   - Else: `dev_set_name(&rdev.wiphy.dev, "phy{idx}")`; on failure free + None.
6. Initialize all locks, lists, work-items (REQ-4 detail).
7. `device_initialize(&rdev.wiphy.dev); dev.class = &IEEE80211_CLASS; dev.platform_data = rdev; device_enable_async_suspend(...)`.
8. `wiphy_net_set(&rdev.wiphy, &INIT_NET)`.
9. `rdev.rfkill_ops.set_block = Cfg80211::rfkill_set_block; rdev.wiphy.rfkill = rfkill_alloc(...)`; on alloc-fail `wiphy_free` + None.
10. Initialize MIB defaults: retry_short=7, retry_long=4, frag_threshold=u32::MAX, rts_threshold=u32::MAX, coverage_class=0, max_num_csa_counters=1, max_sched_scan_plans=1, max_sched_scan_plan_interval=u32::MAX.
11. `Ok(&mut rdev.wiphy)`.

`Cfg80211::wiphy_register(wiphy) -> Result<(), Errno>`:
1. Resolve `rdev = wiphy_to_rdev(wiphy)`.
2. Validate WoWLAN, NAN, NAN_DATA, WDS-reject, PMSR/FTM, regulatory flags, coalesce, AP-SME, MAC-ACL, BSS-select, 4WAY-HSHK, FW-roam: each WARN_ON-and-return-EINVAL on violation (REQ-5).
3. `if wiphy.addresses.is_some()`: `wiphy.perm_addr = addresses[0]`.
4. `wiphy.interface_modes &= ((1 << NUM_IFTYPES) - 1) & !1` (drop reserved bit 0 = UNSPECIFIED).
5. `Cfg80211::verify_combinations(wiphy)?`.
6. For `band` in `NL80211_BAND_2GHZ..NUM_NL80211_BANDS`:
   - If `wiphy.bands[band].is_none()`: continue.
   - `sband.band = band`. `n_channels > 0`.
   - `n_bitrates > 0` except 60GHz/S1GHZ.
   - `band == 6GHZ` ⟹ !ht_supported && !vht_supported.
   - Per-channel: capture orig_flags, orig_mag, orig_mpwr; require `freq_offset < 1000`.
   - Per iftype-data: validate HE/EHT capabilities, AP/non-AP separation.
   - `have_band = true`.
7. `if !have_band ⟹ -EINVAL`.
8. `wiphy_regulatory_register(wiphy)`.
9. `device_add(&wiphy.dev)?`.
10. `wiphy.debugfsdir = debugfs_create_dir(name, IEEE80211_DEBUGFS_DIR)`.
11. `rfkill_register(rfkill)`.
12. Under RTNL: `list_add_rcu(&rdev.list, &CFG80211_RDEV_LIST); cfg80211_rdev_list_generation += 1; rdev.wiphy.registered = true`.
13. `nl80211_notify_wiphy(rdev, NL80211_CMD_NEW_WIPHY)`.
14. `Ok(())`.

`Cfg80211::wiphy_unregister(wiphy)`:
1. `wait_event(rdev.dev_wait, ({ let g = wiphy_lock(&rdev.wiphy); let n = rdev.opencount; drop(g); n == 0 }))`.
2. If `rfkill`: `rfkill_unregister(rfkill)`.
3. RTNL + wiphy_lock: `nl80211_notify_wiphy(rdev, NL80211_CMD_DEL_WIPHY); rdev.wiphy.registered = false; WARN_ON(!wdev_list.is_empty()); debugfs_remove_recursive; list_del_rcu(&rdev.list); synchronize_rcu(); wiphy_regulatory_deregister; cfg80211_rdev_list_generation += 1; device_del(&rdev.wiphy.dev)`.
4. CONFIG_PM: if `wowlan_config && rdev.ops.set_wakeup.is_some()`: `rdev_set_wakeup(rdev, false)`.
5. `cfg80211_process_wiphy_works(rdev, None)`.
6. Cancel/flush every per-rdev work item.
7. `cfg80211_rdev_free_wowlan(rdev); cfg80211_free_coalesce(rdev.coalesce); rdev.coalesce = None`.

`Cfg80211::netdev_notifier_call(nb, state, ptr) -> NotifierRetVal`:
1. Resolve `dev = netdev_notifier_info_to_dev(ptr); wdev = dev.ieee80211_ptr`.
2. `if wdev.is_none()`: return NOTIFY_DONE.
3. `rdev = wiphy_to_rdev(wdev.wiphy)`. WARN if iftype is UNSPECIFIED.
4. Per-state dispatch (REQ-9).
5. `wireless_nlevent_flush(); NOTIFY_OK`.

`Cfg80211::process_wiphy_works(rdev, end)`:
1. `lockdep_assert_held(&rdev.wiphy.mtx)`.
2. `let mut limit = 100;`
3. Loop while `!rdev.wiphy_work_list.is_empty()`:
   - Lock `wiphy_work_lock`.
   - `wk = list_first_entry(&rdev.wiphy_work_list, WiphyWork, entry); list_del_init(&wk.entry); drop lock`.
   - `trace_wiphy_work_run(&rdev.wiphy, wk); (wk.func)(&rdev.wiphy, wk);`
   - Retake lock.
   - If `wk == end`: break.
   - `limit -= 1`; if `limit == 0`: WARN + `INIT_LIST_HEAD(&rdev.wiphy_work_list)`; break.
4. Drop lock.

## Hardening

(Inherits row-1 features from `net/00-overview.md` § Hardening — KASAN, KCSAN, SLUB_DEBUG, RANDSTRUCT, fortify-source.)

cfg80211-specific reinforcement:

- **Per-`wiphy.mtx` held for every `cfg80211_ops` invocation** — defense against driver-races on internal state; lockdep validated via `lockdep_assert_held` at every shim entry.
- **Per-WARN_ON pre-flight checks in `wiphy_new_nm`** — defense against malformed `cfg80211_ops` reaching production (op-pair invariants).
- **Per-`wiphy_register` band/channel/iftype/iface-combination validators** — defense against driver advertising impossible operating modes.
- **Per-6GHz rejecting legacy HT/VHT** — defense against pre-HE drivers attempting 6 GHz operation (FCC violation).
- **Per-WDS iftype rejected** — defense against deprecated 4-address mode resurrecting.
- **Per-regulatory-flag mutual-exclusion** — defense against driver claiming SELF_MANAGED + CUSTOM_REG (ambiguous regdomain ownership).
- **Per-`REGULATORY_WIPHY_SELF_MANAGED` opt-out from global hints** — defense against global CRDA hint overwriting self-managed driver's authoritative regdomain.
- **Per-CRDA signed regdb** (`CONFIG_CFG80211_REQUIRE_SIGNED_REGDB`) — defense against unsigned/malicious country-code blob.
- **Per-`wait_event(opencount==0)` in `wiphy_unregister`** — defense against unregister while any wdev still up.
- **Per-`disable_work_sync` / `flush_work` / `cancel_work_sync` matrix at unregister** — defense against work re-arming during teardown.
- **Per-`process_wiphy_works` runaway-limit (100)** — defense against malicious or buggy driver enqueueing endlessly.
- **Per-`wiphy_work_lock` + list_empty check** — defense against double-enqueue of the same wiphy_work.
- **Per-`netns_immutable = true` on every wireless netdev** — defense against `ip link set wlanX netns N` decoupling a netdev from its wiphy.
- **Per-`cfg80211_pernet_exit` repatriation** — defense against rdev orphaning when its netns dies (forces `WIPHY_FLAG_NETNS_OK` wiphys back to `init_net`).
- **Per-RTNL + per-wiphy.mtx ordering** — defense against deadlock from inverted lock order (documented invariant: RTNL outer, wiphy.mtx inner).
- **Per-`NETDEV_PRE_UP` rfkill check** — defense against bring-up over a blocked radio.
- **Per-`NETDEV_PRE_UP` NAN_DATA dependency** — defense against NAN_DATA without underlying NAN interface.
- **Per-`scan_req->req.wdev == wdev` abort on NETDEV_DOWN** — defense against UAF when wdev frees while driver holds the scan_request.
- **Per-`sched_scan_req_list` per-wdev sweep on NETDEV_DOWN** — defense against sched-scan-for-dead-wdev.
- **Per-`generation` counters (`devlist_generation`, `bss_generation`, `cfg80211_rdev_list_generation`)** — defense against userspace dump tearing (NL80211 returns -ENOBUFS / restart on generation change).
- **Per-`wdev->nl_owner_dead` cleanup via `cfg80211_destroy_iface_wk`** — defense against NL80211-owned vifs surviving owner-process exit.
- **Per-`WIPHY_FLAG_NETNS_OK` opt-in for netns moves** — defense against drivers that lack netns awareness being moved.

## Grsecurity/PaX-style Reinforcement

- **PAX_USERCOPY** — bounds `NL80211_CMD_*` payloads (SSIDs, IEs, scan-req frames, key-material) across the netlink/user boundary.
- **PAX_KERNEXEC** — keeps `cfg80211_ops` dispatch and the regdb signature-verify path text W^X.
- **PAX_RANDKSTACK** — randomises stack per nl80211 syscall entry, defeating ROP under scan/connect floods.
- **PAX_REFCOUNT** — wraps `wiphy_idx`/`wdev` and BSS-entry refcounts to catch UAF on rdev teardown.
- **PAX_MEMORY_SANITIZE** — zeroes WPA/WEP/IGTK/BIP key material in `cfg80211_cached_keys`, GTK rekey buffers, and PMKSA on free.
- **PAX_UDEREF** — protects nl80211 attribute parsers from deref'ing userland on policy mismatch.
- **PAX_RAP / kCFI** — protects indirect dispatch of `cfg80211_ops` shims and per-driver vtables.
- **GRKERNSEC_HIDESYM** — hides `cfg80211_*`, `nl80211_*`, `reg_*` symbols from unprivileged readers.
- **GRKERNSEC_DMESG** — restricts dmesg so regulatory / supplicant debug never leaks BSSIDs / PMKIDs.
- **wiphy ops strict CAP_NET_ADMIN** — every state-mutating `NL80211_CMD_*` (KEY, CONNECT, SET_REG, START_AP, NAN, PMSR) requires CAP_NET_ADMIN in `wiphy->_net`'s user_ns; no fallback for CAP_NET_RAW alone.
- **NL80211_CMD_* attr validation** — strict `NLA_POLICY_*` bounds + length checks reject oversized SSID/IE/PMKID/key material before reaching driver shim.
- **Regulatory db signature** — `CONFIG_CFG80211_REQUIRE_SIGNED_REGDB` is mandatory; unsigned regdb blobs are refused even with `cfg80211.ieee80211_regdom=`.
- **BSS-table PAX_REFCOUNT** — `cfg80211_bss` entries and `request_keys` are refcount-protected against scan/connect UAF.

Rationale: cfg80211 is the single trust-anchor between userland (wpa_supplicant, hostapd, NetworkManager) and every wireless driver in the tree; USERCOPY+UDEREF on the nl80211 path, strict CAP_NET_ADMIN, mandatory regdb signature verification, and MEMORY_SANITIZE on every key cache collapse the cross-driver attack surface to driver-internal bugs alone.
