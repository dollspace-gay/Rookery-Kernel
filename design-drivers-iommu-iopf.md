## Design Specification

### Summary

The **IOPF (IO Page Fault)** framework is the generic plumbing that lets IOMMU drivers (Intel VT-d PRI, AMD-Vi PPR, ARM SMMUv3 PRI) report PCI **PRG (Page Request Group)** faults — generated by endpoints implementing the PCIe PRI extension — up to per-domain page-fault handlers (e.g. SVA + mm hand-off, IOMMUFD eventq, KVM nested-translation faults). Per-IOMMU-driver enqueue: low-level fault IRQ / threaded handler builds a `struct iopf_fault` (which wraps `struct iommu_fault`) and calls `iommu_report_device_fault(dev, evt)`. Per-PRG accumulation: non-`LAST_PAGE` requests are appended to `fault_param->partial`; on the LAST_PAGE, a `struct iopf_group` is allocated, the partial chain is splice-merged into `group->faults`, the group is added to `fault_param->faults`, and the domain's `iopf_handler(group)` is invoked — typically queueing work onto the per-queue UNBOUND workqueue `iopf_queue->wq`. Per-completion: the domain handler must call `iopf_group_response(group, code)` (one of `IOMMU_PAGE_RESP_SUCCESS`, `_INVALID`, `_FAILURE`) which calls `ops->page_response(dev, last_fault, &resp)` to send the PCIe PRG response to the device, then `iopf_free_group(group)`.

Per-device lifecycle: `iopf_queue_add_device` allocates `struct iommu_fault_param`, RCU-publishes it via `dev->iommu->fault_param`; `iopf_queue_remove_device` flushes pending groups (each ack'd with `IOMMU_PAGE_RESP_INVALID`), discards partials, RCU-zeroes the pointer, drops the refcount.

Per-OS-wide: there is exactly one `iopf_queue` per IOMMU driver instance (allocated by `iopf_queue_alloc(name)`); the workqueue is `WQ_UNBOUND` because faults across groups are independent. Critical for: SVA page-fault hand-off into `handle_mm_fault`, IOMMUFD VFIO-PRI passthrough, dependable PRG response timing so endpoints don't stall.

This Tier-3 covers `drivers/iommu/io-pgfault.c` (~549 lines).

### Acceptance Criteria

- [ ] AC-1: Non-LAST_PAGE PRG element added to `fault_param.partial`; LAST_PAGE triggers group build and handler dispatch.
- [ ] AC-2: `iopf_group_alloc` splices partials with matching `grpid` into `group.faults`; partials with mismatched grpid remain in `partial`.
- [ ] AC-3: PASID-valid path: `iommu_attach_handle_get(group, pasid, 0)` succeeds → returned attach used.
- [ ] AC-4: PASID-valid path: attach_get fails AND `ops.user_pasid_table` true → fallback to `IOMMU_DOMAIN_NESTED` attach on `IOMMU_NO_PASID`.
- [ ] AC-5: No fault handler installed → `iopf_error_response` sends `IOMMU_PAGE_RESP_INVALID`; `iommu_report_device_fault` returns -EINVAL.
- [ ] AC-6: Group `kzalloc` failure → uses stack `abort_group`; handler aborted; PRG response = `IOMMU_PAGE_RESP_FAILURE`; `__iopf_free_group` (not `iopf_free_group`) called (no kfree of stack).
- [ ] AC-7: Domain `iopf_handler` returns non-zero → group response = FAILURE; group freed.
- [ ] AC-8: Successful resolution: `iopf_group_response(group, SUCCESS)` invokes `ops.page_response` exactly once; `pending_node` removed.
- [ ] AC-9: Double-response suppression: second `iopf_group_response` after `pending_node` removed is a no-op.
- [ ] AC-10: `iopf_queue_flush_dev`: `flush_workqueue` returns after all queued groups completed.
- [ ] AC-11: `iopf_queue_remove_device`: every pending group ack'd with `IOMMU_PAGE_RESP_INVALID`; partials freed; `fault_param` RCU-zeroed; users-refcount dropped.
- [ ] AC-12: `iopf_queue_add_device` on device with `fault_param` already set → returns -EBUSY without leak.
- [ ] AC-13: `iopf_queue_add_device` on device whose `ops.page_response` is NULL → returns -ENODEV.
- [ ] AC-14: `iopf_queue_discard_partial` clears partial-list for every device under queue.
- [ ] AC-15: `iopf_queue_free` removes all devices then destroys workqueue + kfree(queue).
- [ ] AC-16: Refcount: `iommu_report_device_fault` get + `iopf_free_group` put balanced (no UAF, no leak).

### Architecture

```
struct IopfFault {
  fault: IommuFault,                     // type + prm{flags,pasid,grpid,perm,addr,private[]}
  list: ListNode,
}

struct IopfGroup {
  faults: ListHead<IopfFault>,           // PRG chain, LAST inserted first then partials moved before
  last_fault: IopfFault,                 // inline LAST_PAGE element (NOT heap)
  pending_node: ListNode,                // anchored in fault_param.faults
  fault_param: *IommuFaultParam,         // refcounted handle
  attach_handle: *IommuAttachHandle,
  fault_count: usize,
  cookie: *mut (),                       // domain-handler scratch
}

struct IopfQueue {
  wq: *WorkqueueStruct,                  // alloc_workqueue("iopf_queue/<name>", WQ_UNBOUND, 0)
  devices: ListHead<IommuFaultParam>,    // via queue_list
  lock: Mutex<()>,
}

struct IommuFaultParam {                  // RCU-published as dev->iommu->fault_param
  lock: Mutex<()>,                       // guards partial + faults
  users: Refcount,
  rcu: RcuHead,
  dev: *Device,
  queue: *IopfQueue,
  queue_list: ListNode,                  // in IopfQueue.devices
  partial: ListHead<IopfFault>,          // non-LAST awaiting LAST
  faults: ListHead<IopfGroup>,           // groups in-flight via pending_node
}
```

`Iopf::report_device_fault(dev, evt) -> Result<()>`:
1. attach = Iopf::find_fault_handler(dev, evt).
2. if attach.is_none(): goto err_bad_iopf.
3. param = Iopf::get_dev_fault_param(dev).
4. if WARN_ON(param.is_none()): goto err_bad_iopf.
5. if !(evt.fault.prm.flags & PR_LAST_PAGE):
   - ret = Iopf::report_partial_fault(param, &evt.fault).
   - Iopf::put_dev_fault_param(param).
   - return ret.
6. /* LAST_PAGE */
7. let abort_group = IopfGroup::default() /* on stack */.
8. group = Iopf::group_alloc(param, evt, &abort_group).
9. if group == &abort_group: goto err_abort.
10. group.attach_handle = attach.
11. if attach.domain.iopf_handler(group) != 0: goto err_abort.
12. return Ok(()).
13. err_abort:
    - dev_warn_ratelimited("iopf with pasid {} aborted", evt.fault.prm.pasid).
    - Iopf::group_response(group, IOMMU_PAGE_RESP_FAILURE).
    - if group == &abort_group: Iopf::free_group_inner(group). else: Iopf::free_group(group).
    - return Ok(()).
14. err_bad_iopf:
    - if evt.fault.type == IOMMU_FAULT_PAGE_REQ: Iopf::error_response(dev, evt).
    - return Err(-EINVAL).

`Iopf::find_fault_handler(dev, evt) -> Option<*IommuAttachHandle>`:
1. if evt.fault.prm.flags & PR_PASID_VALID:
   - attach = iommu_attach_handle_get(dev.iommu_group, evt.fault.prm.pasid, 0).
   - if IS_ERR(attach):
     - ops = dev_iommu_ops(dev).
     - if !ops.user_pasid_table: return None.
     - attach = iommu_attach_handle_get(dev.iommu_group, IOMMU_NO_PASID, IOMMU_DOMAIN_NESTED).
     - if IS_ERR(attach): return None.
2. else:
   - attach = iommu_attach_handle_get(dev.iommu_group, IOMMU_NO_PASID, 0).
   - if IS_ERR(attach): return None.
3. if attach.domain.iopf_handler.is_none(): return None.
4. return Some(attach).

`Iopf::report_partial_fault(param, fault) -> Result<()>`:
1. iopf = kzalloc(IopfFault); if !iopf: return Err(-ENOMEM).
2. iopf.fault = *fault.
3. mutex_lock(&param.lock).
4. list_add(&iopf.list, &param.partial).
5. mutex_unlock(&param.lock).
6. return Ok(()).

`Iopf::group_alloc(param, evt, abort_group) -> *IopfGroup`:
1. group = kzalloc(IopfGroup); if !group: group = abort_group.
2. group.fault_param = param.
3. group.last_fault.fault = evt.fault.
4. list_init(&group.faults); list_init(&group.pending_node).
5. list_add(&group.last_fault.list, &group.faults).
6. mutex_lock(&param.lock).
7. for iopf in list_for_each_entry_safe(&param.partial):
   - if iopf.fault.prm.grpid == evt.fault.prm.grpid:
     - list_move(&iopf.list, &group.faults). /* into THIS group */
8. list_add(&group.pending_node, &param.faults).
9. mutex_unlock(&param.lock).
10. group.fault_count = list_count_nodes(&group.faults).
11. return group.

`Iopf::group_response(group, status)`:
1. param = group.fault_param.
2. iopf = &group.last_fault.
3. dev = param.dev.
4. ops = dev_iommu_ops(dev).
5. resp = IommuPageResponse { pasid: iopf.fault.prm.pasid, grpid: iopf.fault.prm.grpid, code: status }.
6. mutex_lock(&param.lock).
7. if !list_empty(&group.pending_node):
   - ops.page_response(dev, &group.last_fault, &resp).
   - list_del_init(&group.pending_node).
8. mutex_unlock(&param.lock).

`Iopf::free_group(group)`:
1. Iopf::free_group_inner(group):
   - for iopf in list_for_each_entry_safe(&group.faults):
     - if !(iopf.fault.prm.flags & PR_LAST_PAGE): kfree(iopf).
   - Iopf::put_dev_fault_param(group.fault_param).
2. kfree(group).

`Iopf::get_dev_fault_param(dev) -> Option<*IommuFaultParam>`:
1. rcu_read_lock.
2. fault_param = rcu_dereference(dev.iommu.fault_param).
3. if fault_param ∧ !refcount_inc_not_zero(&fault_param.users): fault_param = NULL.
4. rcu_read_unlock.
5. return fault_param.

`Iopf::put_dev_fault_param(fault_param)`:
1. if refcount_dec_and_test(&fault_param.users): kfree_rcu(fault_param, rcu).

`Iopf::queue_alloc(name) -> Option<*IopfQueue>`:
1. queue = kzalloc(IopfQueue); if !queue: return None.
2. queue.wq = alloc_workqueue("iopf_queue/{}", WQ_UNBOUND, 0, name).
3. if !queue.wq: kfree(queue); return None.
4. list_init(&queue.devices); mutex_init(&queue.lock).
5. return Some(queue).

`Iopf::queue_add_device(queue, dev) -> Result<()>`:
1. ops = dev_iommu_ops(dev).
2. if ops.page_response.is_none(): return Err(-ENODEV).
3. mutex_lock(&queue.lock); mutex_lock(&dev.iommu.lock).
4. if rcu_dereference_check(dev.iommu.fault_param, held(param.lock)): return Err(-EBUSY).
5. fault_param = kzalloc(IommuFaultParam); if !fault_param: return Err(-ENOMEM).
6. mutex_init(&fault_param.lock).
7. list_init(&fault_param.faults); list_init(&fault_param.partial).
8. fault_param.dev = dev; fault_param.queue = queue.
9. refcount_set(&fault_param.users, 1).
10. list_add(&fault_param.queue_list, &queue.devices).
11. rcu_assign_pointer(dev.iommu.fault_param, fault_param).
12. mutex_unlock(both).
13. return Ok(()).

`Iopf::queue_remove_device(queue, dev)`:
1. mutex_lock(&queue.lock); mutex_lock(&dev.iommu.lock).
2. fault_param = rcu_dereference_check(dev.iommu.fault_param, ...).
3. if WARN_ON(!fault_param ∨ fault_param.queue != queue): unlock.
4. mutex_lock(&fault_param.lock).
5. for partial in list_for_each_entry_safe(&fault_param.partial): kfree(partial).
6. for group in list_for_each_entry_safe(&fault_param.faults, pending_node):
   - resp = { pasid, grpid, code: IOMMU_PAGE_RESP_INVALID }.
   - ops.page_response(dev, &group.last_fault, &resp).
   - list_del_init(&group.pending_node).
   - Iopf::free_group(group).
7. mutex_unlock(&fault_param.lock).
8. list_del(&fault_param.queue_list).
9. rcu_assign_pointer(dev.iommu.fault_param, NULL).
10. Iopf::put_dev_fault_param(fault_param). /* drop add_device ref */
11. mutex_unlock(both).

`Iopf::queue_flush_dev(dev) -> Result<()>`:
1. param = rcu_dereference_check(dev.iommu.fault_param, true).
2. if WARN_ON(!param): return Err(-ENODEV).
3. flush_workqueue(param.queue.wq).
4. return Ok(()).

`Iopf::queue_discard_partial(queue) -> Result<()>`:
1. if !queue: return Err(-EINVAL).
2. mutex_lock(&queue.lock).
3. for param in list_for_each_entry(&queue.devices):
   - mutex_lock(&param.lock).
   - for iopf in list_for_each_entry_safe(&param.partial):
     - list_del(&iopf.list); kfree(iopf).
   - mutex_unlock(&param.lock).
4. mutex_unlock(&queue.lock).
5. return Ok(()).

`Iopf::queue_free(queue)`:
1. if !queue: return.
2. for param in list_for_each_entry_safe(&queue.devices, queue_list):
   - Iopf::queue_remove_device(queue, param.dev).
3. destroy_workqueue(queue.wq).
4. kfree(queue).

### compatibility contract

REQ-1: struct iopf_fault:
- fault: `struct iommu_fault` — `{ type, padding, prm }`.
  - prm: `struct iommu_fault_page_request` — `{ flags, pasid, grpid, perm, addr, private_data[] }`.
  - prm.flags bits: `IOMMU_FAULT_PAGE_REQUEST_PASID_VALID`, `_LAST_PAGE`, `_PRIV_DATA`.
- list: `list_head` — linked into `fault_param->partial` or `group->faults`.

REQ-2: struct iopf_group:
- faults: list_head — head of LAST_PAGE-terminated PRG chain (most-recent-first; LAST_PAGE inserted via list_add then partials moved before it).
- last_fault: `struct iopf_fault` — the LAST_PAGE entry (stored inline in group; not heap-allocated).
- pending_node: list_head — anchored in `fault_param->faults` while handler in-flight.
- fault_param: *IommuFaultParam — refcounted handle to per-device context.
- attach_handle: *IommuAttachHandle — resolved domain attach.
- fault_count: usize — list_count_nodes(&faults).
- cookie: void * — per-domain handler scratch.

REQ-3: struct iopf_queue:
- wq: workqueue_struct * — `alloc_workqueue("iopf_queue/%s", WQ_UNBOUND, 0, name)`.
- devices: list_head — devices producing into this queue (linked via `fault_param->queue_list`).
- lock: mutex — guards `devices`.

REQ-4: struct iommu_fault_param (per `dev->iommu->fault_param`, RCU-published):
- lock: mutex — guards `faults` + `partial`.
- users: refcount_t — refcount; freed via `kfree_rcu`.
- rcu: rcu_head — RCU teardown anchor.
- dev: *Device.
- queue: *IopfQueue.
- queue_list: list_head — linked into `queue->devices`.
- partial: list_head — non-LAST_PAGE PRG elements awaiting LAST_PAGE.
- faults: list_head — in-flight groups (linked via `group->pending_node`).

REQ-5: iommu_report_device_fault(dev, evt):
- attach_handle = find_fault_handler(dev, evt).
- if !attach_handle: goto err_bad_iopf.
- iopf_param = iopf_get_dev_fault_param(dev).
- if WARN_ON(!iopf_param): goto err_bad_iopf.
- if !(evt.fault.prm.flags & IOMMU_FAULT_PAGE_REQUEST_LAST_PAGE):
  - ret = report_partial_fault(iopf_param, &evt.fault).
  - iopf_put_dev_fault_param(iopf_param).
  - return ret.
- /* LAST_PAGE: build group */
- group = iopf_group_alloc(iopf_param, evt, &abort_group).
- if group == &abort_group: goto err_abort.
- group.attach_handle = attach_handle.
- /* Hand to domain. On success handler must call iopf_group_response + iopf_free_group. */
- if attach_handle.domain.iopf_handler(group): goto err_abort.
- return 0.
- err_abort:
  - dev_warn_ratelimited("iopf with pasid %d aborted", fault.prm.pasid).
  - iopf_group_response(group, IOMMU_PAGE_RESP_FAILURE).
  - if group == &abort_group: __iopf_free_group(group). else: iopf_free_group(group).
  - return 0.
- err_bad_iopf:
  - if evt.fault.type == IOMMU_FAULT_PAGE_REQ: iopf_error_response(dev, evt).
  - return -EINVAL.

REQ-6: iopf_get_dev_fault_param(dev):
- rcu_read_lock.
- fault_param = rcu_dereference(dev.iommu.fault_param).
- if fault_param ∧ !refcount_inc_not_zero(&fault_param.users): fault_param = NULL.
- rcu_read_unlock.
- return fault_param.

REQ-7: iopf_put_dev_fault_param(fault_param):
- if refcount_dec_and_test(&fault_param.users): kfree_rcu(fault_param, rcu).

REQ-8: report_partial_fault(fault_param, fault):
- iopf = kzalloc(struct iopf_fault).
- if !iopf: return -ENOMEM.
- iopf.fault = *fault.
- mutex_lock(&fault_param.lock).
- list_add(&iopf.list, &fault_param.partial).
- mutex_unlock(&fault_param.lock).
- return 0.

REQ-9: iopf_group_alloc(iopf_param, evt, abort_group):
- group = kzalloc(struct iopf_group).
- if !group: group = abort_group.
- group.fault_param = iopf_param.
- group.last_fault.fault = evt.fault.
- INIT_LIST_HEAD(&group.faults); INIT_LIST_HEAD(&group.pending_node).
- list_add(&group.last_fault.list, &group.faults).
- /* Splice partials with matching grpid */
- mutex_lock(&iopf_param.lock).
- for iopf, next in list_for_each_entry_safe(iopf_param.partial):
  - if iopf.fault.prm.grpid == evt.fault.prm.grpid:
    - list_move(&iopf.list, &group.faults).
- list_add(&group.pending_node, &iopf_param.faults).
- mutex_unlock(&iopf_param.lock).
- group.fault_count = list_count_nodes(&group.faults).
- return group.

REQ-10: find_fault_handler(dev, evt):
- if evt.fault.prm.flags & IOMMU_FAULT_PAGE_REQUEST_PASID_VALID:
  - attach_handle = iommu_attach_handle_get(dev.iommu_group, evt.fault.prm.pasid, 0).
  - if IS_ERR(attach_handle):
    - ops = dev_iommu_ops(dev).
    - if !ops.user_pasid_table: return NULL.
    - /* Fall back: user-managed PASID table → route via nested domain on RID */
    - attach_handle = iommu_attach_handle_get(dev.iommu_group, IOMMU_NO_PASID, IOMMU_DOMAIN_NESTED).
    - if IS_ERR(attach_handle): return NULL.
- else:
  - attach_handle = iommu_attach_handle_get(dev.iommu_group, IOMMU_NO_PASID, 0).
  - if IS_ERR(attach_handle): return NULL.
- if !attach_handle.domain.iopf_handler: return NULL.
- return attach_handle.

REQ-11: iopf_error_response(dev, evt):
- ops = dev_iommu_ops(dev).
- resp = { pasid: evt.fault.prm.pasid, grpid: evt.fault.prm.grpid, code: IOMMU_PAGE_RESP_INVALID }.
- ops.page_response(dev, evt, &resp).

REQ-12: iopf_group_response(group, status):
- fault_param = group.fault_param.
- iopf = &group.last_fault.
- dev = fault_param.dev.
- ops = dev_iommu_ops(dev).
- resp = { pasid: iopf.fault.prm.pasid, grpid: iopf.fault.prm.grpid, code: status }.
- /* Only respond if still pending (i.e. queue_remove_device hasn't drained) */
- mutex_lock(&fault_param.lock).
- if !list_empty(&group.pending_node):
  - ops.page_response(dev, &group.last_fault, &resp).
  - list_del_init(&group.pending_node).
- mutex_unlock(&fault_param.lock).

REQ-13: iopf_free_group(group):
- __iopf_free_group(group):
  - for iopf, next in list_for_each_entry_safe(group.faults):
    - if !(iopf.fault.prm.flags & IOMMU_FAULT_PAGE_REQUEST_LAST_PAGE): kfree(iopf).
    - /* The LAST_PAGE iopf is `group->last_fault` (inline) — must not kfree */
  - iopf_put_dev_fault_param(group.fault_param). /* pair iommu_report_device_fault */
- kfree(group).

REQ-14: iopf_queue_flush_dev(dev):
- iopf_param = rcu_dereference_check(dev.iommu.fault_param, true).
- if WARN_ON(!iopf_param): return -ENODEV.
- flush_workqueue(iopf_param.queue.wq).
- return 0.

REQ-15: iopf_queue_discard_partial(queue):
- if !queue: return -EINVAL.
- mutex_lock(&queue.lock).
- for iopf_param in list_for_each_entry(queue.devices):
  - mutex_lock(&iopf_param.lock).
  - for iopf, next in list_for_each_entry_safe(iopf_param.partial):
    - list_del(&iopf.list); kfree(iopf).
  - mutex_unlock(&iopf_param.lock).
- mutex_unlock(&queue.lock).
- return 0.

REQ-16: iopf_queue_add_device(queue, dev):
- ops = dev_iommu_ops(dev).
- if !ops.page_response: return -ENODEV.
- mutex_lock(&queue.lock); mutex_lock(&param.lock).
- if rcu_dereference_check(param.fault_param, lockdep_is_held(&param.lock)): ret = -EBUSY; unlock.
- fault_param = kzalloc(struct iommu_fault_param).
- if !fault_param: ret = -ENOMEM; unlock.
- mutex_init(&fault_param.lock).
- INIT_LIST_HEAD(&fault_param.faults); INIT_LIST_HEAD(&fault_param.partial).
- fault_param.dev = dev.
- refcount_set(&fault_param.users, 1).
- list_add(&fault_param.queue_list, &queue.devices).
- fault_param.queue = queue.
- rcu_assign_pointer(param.fault_param, fault_param).
- unlock; return ret.

REQ-17: iopf_queue_remove_device(queue, dev):
- mutex_lock(&queue.lock); mutex_lock(&param.lock).
- fault_param = rcu_dereference_check(param.fault_param, lockdep_is_held(&param.lock)).
- if WARN_ON(!fault_param ∨ fault_param.queue != queue): unlock.
- mutex_lock(&fault_param.lock).
- /* Drop all partials */
- for partial_iopf, next in list_for_each_entry_safe(fault_param.partial): kfree(partial_iopf).
- /* Ack every outstanding group with INVALID */
- for group, temp in list_for_each_entry_safe(fault_param.faults, pending_node):
  - iopf = &group.last_fault.
  - resp = { pasid: iopf.fault.prm.pasid, grpid: iopf.fault.prm.grpid, code: IOMMU_PAGE_RESP_INVALID }.
  - ops.page_response(dev, iopf, &resp).
  - list_del_init(&group.pending_node).
  - iopf_free_group(group).
- mutex_unlock(&fault_param.lock).
- list_del(&fault_param.queue_list).
- rcu_assign_pointer(param.fault_param, NULL).
- iopf_put_dev_fault_param(fault_param). /* drop add_device ref */
- unlock.

REQ-18: iopf_queue_alloc(name):
- queue = kzalloc(struct iopf_queue).
- queue.wq = alloc_workqueue("iopf_queue/%s", WQ_UNBOUND, 0, name).
- if !queue.wq: kfree(queue); return NULL.
- INIT_LIST_HEAD(&queue.devices); mutex_init(&queue.lock).
- return queue.

REQ-19: iopf_queue_free(queue):
- for iopf_param, next in list_for_each_entry_safe(queue.devices, queue_list):
  - iopf_queue_remove_device(queue, iopf_param.dev).
- destroy_workqueue(queue.wq).
- kfree(queue).

REQ-20: PRG response codes (uapi `enum iommu_page_response_code`):
- `IOMMU_PAGE_RESP_SUCCESS` — fault resolved; retry permitted.
- `IOMMU_PAGE_RESP_INVALID` — request invalid; device must not retry.
- `IOMMU_PAGE_RESP_FAILURE` — fatal; device may take vendor-specific recovery.

REQ-21: PCI PASID Stop Marker discard:
- A Stop Marker (PCIe r4 §10.4.1.2; LRW = 0b100) does not expect a response.
- IOMMU drivers MUST discard Stop Markers before calling `iommu_report_device_fault` — the framework does not handle them.

REQ-22: User-PASID-table fallback:
- When PASID-valid lookup fails AND `ops.user_pasid_table` is true (user owns PASID table — IOMMUFD-NESTED case): re-attempt via `iommu_attach_handle_get(group, IOMMU_NO_PASID, IOMMU_DOMAIN_NESTED)`. The nested domain handles PASID dispatch in software.

REQ-23: Workqueue policy:
- `WQ_UNBOUND` — faults from different groups are independent (per-group ordering only matters within a group, and the low-level driver enqueues by group before calling the framework).

REQ-24: RCU pointer publication:
- `dev->iommu->fault_param` is RCU-protected. Readers use `rcu_read_lock` + `rcu_dereference`. Writers serialize via `param->lock` and use `rcu_assign_pointer`.
- `kfree_rcu(fault_param, rcu)` defers free until RCU grace-period.

REQ-25: Lock ordering:
- queue.lock > param.lock > fault_param.lock (per `iopf_queue_add_device` / `iopf_queue_remove_device`).
- `iopf_param.lock` alone for partial-list / faults-list mutation in fast path.

REQ-26: Per-IOMMU-driver consumers:
- VT-d: `drivers/iommu/intel/prq.c` calls `iommu_report_device_fault`.
- AMD-Vi: `drivers/iommu/amd/iommu.c` (PPR log handler).
- ARM SMMUv3: `drivers/iommu/arm/arm-smmu-v3/arm-smmu-v3.c` (PRI queue handler).
- All implement `ops.page_response` for PRG response transmission.

REQ-27: IOMMUFD eventq integration:
- IOMMUFD-IO_PAGE_REQ events reach userspace via `drivers/iommu/iommufd/eventq.c`; the domain's `iopf_handler` queues into the eventq, userspace responds via `IOMMU_PAGE_RESPONSE_IOCTL`, which eventually calls `iopf_group_response`.

REQ-28: SVA integration:
- SVA bind installs a `iommu_sva_iopf_handler` (per drivers/iommu/iommu-sva.c) which calls `handle_mm_fault` on the bound `mm` and translates the result into `IOMMU_PAGE_RESP_SUCCESS` / `_INVALID`.

### Layer 2: TLA+

`drivers/iommu/iopf.tla`:
- States: REPORT → (PARTIAL | LAST) ; LAST → ALLOC → SPLICE → HANDLER → RESPONSE → FREE.
- Properties:
  - `safety_partial_then_last_yields_group` — per-PRG: partials with grpid g + LAST(grpid=g) → group with all partials.
  - `safety_no_response_before_handler` — per-LAST: handler dispatch precedes group_response.
  - `safety_remove_acks_all_pending` — per-queue_remove_device: every group in fault_param.faults ack'd with INVALID.
  - `safety_flush_drains_workqueue` — per-queue_flush_dev: flush_workqueue completes only when wq empty.
  - `safety_rcu_no_use_after_free` — per-fault_param: readers via rcu_dereference never see freed pointer.
  - `liveness_per_last_eventually_responded` — per-LAST: eventually iopf_group_response invoked (success or abort).
  - `liveness_per_remove_completes` — per-queue_remove_device: terminates with empty faults+partial.

| Invariant | Component |

### Layer 4: Verus/Creusot functional

`Per-PRG arrival → IOMMU-driver iommu_report_device_fault → partial accumulation OR LAST_PAGE group build → attach_handle resolution (PASID / user-PASID-table / no-PASID) → domain.iopf_handler dispatch → workqueue process → iopf_group_response → ops.page_response (PRG response) → iopf_free_group` semantic equivalence with PCIe r6.0 §10.4 (PRI), Intel VT-d 4.1 §7.5 (Page Request Service), and AMD-Vi §5.2.4 (PPR). Spec source: Documentation/userspace-api/iommufd.rst, Documentation/iommu/sva.rst.

### hardening

(Inherits row-1 features from `drivers/iommu/00-overview.md` § Hardening.)

IOPF reinforcement:

- **Per-`fault_param` RCU + refcount** — defense against per-device-unbind UAF while ISR holds pointer.
- **Per-LAST_PAGE inline storage (`group->last_fault`)** — defense against per-double-free of the LAST iopf.
- **Per-`abort_group` on group `kzalloc` failure** — defense against per-OOM-driven endpoint hang (always emits FAILURE PRG response).
- **Per-`pending_node` gate on `iopf_group_response`** — defense against per-double-response / response-after-remove.
- **Per-`grpid` partition in `iopf_group_alloc`** — defense against per-cross-group fault leakage.
- **Per-`IOMMU_PAGE_RESP_INVALID` flush in `iopf_queue_remove_device`** — defense against per-device-stop dangling PRG (endpoint would retry indefinitely).
- **Per-`flush_workqueue` before PASID free** — defense against per-stale-fault-into-new-mm (TLB-shootdown analogue).
- **Per-Stop-Marker discard requirement (driver-side)** — defense against per-spurious-no-response PRG.
- **Per-`ops.page_response` mandatory at `queue_add_device`** — defense against per-driver missing response path.
- **Per-RCU teardown via `kfree_rcu`** — defense against per-concurrent-reader UAF.
- **Per-`WARN_ON` on fault-param mismatch in `queue_remove_device`** — defense against per-cross-queue mis-removal.
- **Per-WQ_UNBOUND policy explicit** — defense against per-CPU-bound starvation of fault-handling under load.
- **Per-rate-limited abort log (`dev_warn_ratelimited`)** — defense against per-fault-storm log-flood.
